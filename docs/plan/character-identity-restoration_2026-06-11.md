## 1. 배경·진단 (확정)

**증상**: 원격 서버(achat) 26개 캐릭터의 반응·대사가 밋밋하다.

**근본 원인** (다단계 추적으로 확정):
- `persona-codex`로 너+코덱스가 캐릭터 알맹이(성격·욕망·성적 특성·말투)를 깊이 설계했으나, **원격 등록(`02_prompt.md` → DB) 단계에서 그 알맹이가 누락**됐다.
- 등록물엔 외모·관계·이미지/상태창/모드 같은 **시스템 규칙만 과잉**, 캐릭터 알맹이는 결핍 = **"규칙으로 꽉 찬 빈 껍데기"**.
- 멀티캐릭터 작품에서 **서브 캐릭터(언니·장모·처제 등) 성격·성적 특성이 백지** → AI가 전원 평균값으로 굴려 차별화 0.

**입증**:
- 전수 스캔(26개): 성격 키워드 대부분 1~10개, 규칙 키워드 평균 ~45개. wife-us-sergeant = 성격1·성적3·규칙50.
- `01_concept.md`엔 알맹이가 살아있음: next-door-slut(캐릭터별 체형·욕망 톤·행위 언어 484줄), villain-bureau(욕망 아키타입 7그룹·욕망 시그니처 양식), scary-noonas(5인 차별화표). → **설계는 했고 등록서 버렸다.**
- 초기 "심리 과잉" 진단은 **반대였음**(과잉이 아니라 결핍). 로컬 작성문서만 보고 오판 → 원격 실물 확인으로 정정.

## 2. 기준 — babechat 실측 표준 3종 (정착 대상)

원격과 무관한 babechat-studio 작업물(폐기된 76개 character.md)이 아니라, 그 **실측 가이드**(`docs/references/`)에서 도출한 표준. 3개 에이전트 정독으로 확정:

### 2-A. 성적 프로필 표준 (9필드 + 12성향)
- 9필드: 성향유형 / 갭(평소↔음란) / 신체 트레이드마크 / 민감대·약점 / 능동성 모델 / 무너지는 포인트 / 체위·플레이 선호 / 말투 마커 / 관계
- 12성향: 사투리갭모에·표준어애교·혐관굴복NTL·츤데레·얀데레집착·쿨도도·순종순애·누나연상·순진첫경험·유부녀NTL·건방도발·청순반전
- 메인=9필드 풀 / 서브=압축 4필드(성향+갭+말투+무너짐or트레이드마크)
- 진도: first_done 이진 분기(회차 카운터 폐지), 능동성은 긍정형 OPEN 풀로

### 2-B. 성격·매력 표준 (7필드 + 첫턴만개 + 긍정형규칙)
- 7필드: 한줄각인 / 표면성격 / 내면욕구(표면과 갭) / 갭 트리거 / 능동방식 / 말투·말버릇 / 매력 시연 2개
- 첫 턴 만개: 매력을 💭(속마음)에 가두지 말고 **표면 대사·행동**으로. first_mes에 훅(딜레마/질문)+선택지
- 긍정형 규칙: "하지 마라"(억압) → "이렇게 한다"(활성). 한 문장 금지어 3개+ = 리팩터
- 서브: 5필드(메인과 욕구축·능동계열·말투 음색 3축 분화)

### 2-C. 멀티 시스템·로어 표준 (3계층 + 무대3인룰)
- 3계층: 메인 1~2명(description 캐시) / 서브 3~6명(키워드 로어 1인1블록) / 엑스트라(한 줄)
- 무대 3인 룰: 등록은 N명, 동시 깊은 묘사는 3명. 키워드 로어 keys에 [이름+별칭+호칭+결속장소]
- 서브 6요소 블록(250~400자): 외형 / 성격핵 1문장 / 성적프로필(차별화) / 관계(캐릭터간 포함) / 등장트리거 / 말투
- **진행 동력·수치 시스템(호감도·게이지 등): 원본 유지** (사용자 확정 — 이번 스코프 아님. 각 카드 기존 시스템 그대로 두고 **알맹이(성격·성적·말투) 복구에만 집중**. 수치 통일·단계제 전환 안 함)
- 상태창: 헤더 + 무대 캐릭터 카드(최대 3), 서브는 단축 표시 — 단 기존 상태창 구조는 보존

→ **이 3종을 `.claude/skills/persona-codex/references/` 에 표준 문서로 정착** (스킬·페르소나가 참조).
