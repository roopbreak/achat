## FAIL (수정 필수)

### F1. [C / E] 상시 로어 10개 전부 insertion_order=100 — priority 설계가 죽은 코드 🔴 최우선
`lib/db.mjs` `getConstantLore`: `SELECT * FROM lore_entries WHERE constant=1 ... ORDER BY insertion_order`. priority 컬럼은 정렬에 안 쓰인다. 이 스토리 상시 로어 10개는 priority가 75~100으로 다양하지만 **insertion_order가 전부 100** — 결과적으로 SQLite의 암묵 정렬(rowid=id 순, 1264~1273)에 의존한다. 작성자가 "이미지 시스템 priority 100, 자취방 공간 priority 75"로 위계를 설계했지만 엔진에서 그 위계는 작동하지 않는다. 토큰 압박으로 상시 로어가 일부 잘릴 때(`context-builder.mjs`는 상시 로어를 block3에 통째로 넣지만, RECENT_TURNS 축소가 걸리면 전체 시스템이 압박받음) 어떤 게 살아남을지 통제 불가.
- **수정안**: 상시 로어 insertion_order를 priority 의도대로 재배치 — 핵심 시스템(이미지 URL·INFO 박스·합의 안전장치·성적 어휘·응답 우선순위)을 낮은 io(10·20·30...)로, 보조(자취방 공간·페이즈 정의 등)를 높은 io로. **단 F2로 상시 로어 개수 자체를 줄이는 게 우선** — 줄인 뒤 io 재배치.

### F2. [C / E] 상시 로어 10개 — qa-checklist "4개+ FAIL" 기준의 2.5배, 캐시 블록 비대 🔴 최우선
qa-checklist D: "상시 로어 4개 이상 FAIL". 이 스토리는 10개. content 합계 5174자가 매 턴 무조건 캐시 블록(block 3)에 주입된다. description 5672자(block 2) + post_history 1424자(block 6) + first_mes와 합치면 system block이 1.4만 자를 넘는다. `context-builder.mjs`의 RECENT_TURNS 동적 축소(systemTokens가 크면 최근 대화를 깎음)가 일찍 걸려 장기 세션에서 맥락이 손실된다. 멀티 캐릭터 3인 + 페이즈 + 외부 이미지라 시스템이 복잡한 건 사실이나, 10개 중 상당수는 통합·키워드화 가능하다.
- **수정안**: 상시 로어를 **5~6개로 축소**:
  - 통합: `[1264] 이미지 URL 시스템` + `[1265] 상황 코드 매핑` → 1개 (둘 다 이미지 시스템). `[1267] 응답 우선순위 6단계` + `[1269] 페이즈 시스템 정의` → 1개 (응답 순서와 페이즈는 인접). 
  - 키워드화: `[1268] 자취방 공간`은 P2 이후 거의 무관 → 키워드 로어(`침대/소파/화장실/자취방/부엌`)로 전환. `[1265]`의 체위 코드 44~94 상세는 P3 전용이므로 별도 키워드 로어(`체위/정상위/후배위/...`)로 분리하고 상시엔 1~43만.
  - 유지(상시 필수): 이미지+코드 통합 / INFO 박스 형식 / 응답+페이즈 통합 / 합의 안전장치 / 성적 어휘 / 호칭 매트릭스 = 6개.

### F3. [D / E·K] `[1281] 다해 술 회복` — 1글자 범용 키워드 오매칭, P3 진입 게이트 로어가 죽음 🔴 최우선
`keys = ["정신","토","토하","토했","물","깨","멀쩡","술 깨","안정","회복","진정","가라앉","추슬"]`. `context-builder.mjs`의 `keywordMatch`는 순수 substring `includes()`다. `토`는 "**토**요일"·"멘**토**"·"투정"... `물`은 "**물**어봐"·"진짜**물**"·"선**물**"... `깨`는 "**깨**닫"·"**깨**끗"... 무관한 단어에 매번 오매칭된다. 반대로 `토하`·`토했`는 정작 입력에 그 활용형 그대로 안 나오면 누락. 이 로어는 **다해 P3 진입의 안전 게이트**("이 단계 통과 전에는 다해와 어떤 성적 접촉도 시작되지 않음")인데, 트리거가 통제 불능이면 만취 NSFW 가드가 무력화된다.
- **수정안**: 1글자·범용 키 전부 제거. 명사형·복합형으로 교체 — `["정신 차", "정신차", "술 깨", "술깨", "토했", "토하", "토 다", "물 마", "물마", "멀쩡", "괜찮아졌", "진정", "가라앉"]` 식으로 최소 2글자 이상 + 다해 술 회복 맥락에서만 등장하는 표현으로. scan_depth는 4 유지(P1 직후 트리거되므로).

### F4. [D / E] 키워드 로어 위험키 다발 — 6개 항목에서 오매칭 키 발견
- `[1276] 촬영·라이브` `스토리`·`올리`·`찍자`·`셀카` → "스토리" 텍스트 자체가 INFO 박스·로어에 빈출, "올리" = "올리브"·"떠올리"
- `[1277] 잠든 상태` `잠`·`코고` → "**잠**깐"·"**잠**시"에 매번 오매칭. 다해 술 회복 로어와 함께 P1 내내 오작동
- `[1278] 다해 라이프스타일` `모델`·`피팅` / `[1279] 아리` `클럽`·`파티` / `[1280] 시하` `셀럽` → 셋의 직업 키워드가 서로 겹치고 범용
- `[1282] 인플루언서 디바이스` `메시지`·`스토리`·`DM` → `[1276]`과 키워드 충돌, "메시지"는 범용
- **수정안**: 1글자·범용어 제거, 복합 명사형으로. `잠` → `잠들`·`곯아떨어`·`잠들었`, `스토리` → `스토리 올`·`스토리 켜`, `모델` → `다해 모델`·`피팅 일`, `메시지` → `별그램 메시지`·`디엠 알림`. `[1276]`/`[1282]` 키워드 중복(`스토리`)을 제거하고 역할 분리: `[1276]`=촬영 행위 트리거, `[1282]`=디엠 우연 노출.

### F5. [G / N] 아리·시하 NSFW 행위 차별화가 "신음 한 줄"로만 존재 — 엔진 미작동 위험
