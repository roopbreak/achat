## 종합 판정: **FAIL**

FAIL 7건 (A 3 · E 3 · K 1), WARN 다수. 핵심 원인은 **페이즈 시스템 전체가 "Day(날짜) 카운팅"에 묶여 있다**는 것 — AChat 엔진의 "턴 카운팅 금지" 제약을 정면으로 위반하며, 그 결과 일정 로어 10개가 죽고 페이즈 전환이 망가진다.

> **하네스 검증 메모**: 작성 시점 QA(이전 03_qa_report.md)는 동일 스토리를 **PASS**로 판정하고 "페이즈 시스템이 가이드라인을 완벽히 준수, 턴 수 금지"라고 칭찬했다. 새 6인 하네스는 정반대를 발견했다 — 적대적 E 페르소나가 `lib/context-builder.mjs`의 substring 매칭을 코드로 확인해 일정 로어 10개가 트리거 불가임을 확증했고, A·E·P·D 4인이 Day 기반 자동 전환을 독립적으로 같은 결함으로 지목했다. 다관점 + 적대적 검수가 단일 시각의 사각지대를 메운 사례.

## FAIL (수정 필수)

### F1. [C·H / A·E·P·D] Day 기반 자동 페이즈 전환 — 엔진 "턴 카운팅 금지" 위반 🔴 최우선
post_history에 `Phase 1 (Day 1~3)` / "Day 4 이상 도달 시 자동 Phase 2 전환" / "Day 7 이상 자동 Phase 3" / "Day 9 밤 강제 전환"이 박혀 있다. AChat 엔진은 AI가 턴·날짜를 세지 못한다(achat-engine.md §8). 게다가 "날짜 전환은 {{user}}가 명시적으로 지시할 때만"이라는 조항과 **자기모순** — 유저가 날짜를 안 올리면 Phase가 영영 1에 머물고, 점프하면 Phase가 튄다. 페이즈 명칭의 `(Day N~N)` 괄호 때문에 AI가 스테이터스 Day를 보고 페이즈를 강제 동기화 → 유저 행동 기반 전환 원칙이 무력화.
- **수정안**: 페이즈를 날짜에서 완전 분리. 명칭에서 `(Day N~N)` 제거. "Day N 이상 자동 전환" 조항 전부 삭제. 전환은 순수 유저 행동 트리거로만. 시간 압박은 "Day가 7+ 이면 시아 속마음·대사에 조바심 강화"라는 **연출 강도 지시**로 (페이즈를 강제 이동시키지 않음).

### F2. [C·H / A·E] "Day 9 밤 강제 전환" — 트리거 시점을 AI가 못 잡음
post_history와 로어 [1116]에 중복 기재. 날짜 추적 불가 + 매 턴 재주입되는 post_history에 일회성 이벤트를 박아 둠 → 매 턴 트리거되거나 영영 안 됨.
- **수정안**: 날짜 트리거 → **상태 기반**으로. "Phase 3에 도달했고 + 유저가 마지막 밤/귀국 전날 장면으로 진입했으며 + 아직 성관계가 없으면 → 시아가 그 장면에서 직접 덮친다." 로어 [1116]도 동일하게 수정 (키워드 `마지막밤` 활용).

### F3. [D / E] Day 1~10 일정 로어 10개가 죽은 로어 — 키워드 트리거 불가 🔴 최우선
`context-builder.mjs`의 `keywordMatch`는 순수 substring `includes()`. 일정 로어 키워드가 `day1, 1일차, 첫째날` 등인데, 실제 채팅에서 이 문자열이 입력/출력에 그대로 등장하지 않는다. 스테이터스 표기는 `📅 Day: N/10` 형식이라 `day1`과 매칭 안 됨. scan_depth 4 → 10개 일정 로어 전부 주입 안 됨, AI가 일정을 임의 생성.
- **수정안 (택1)**: ① 키를 스테이터스 표기와 일치시킴 — `["Day: 1/", "1일차", "왓아룬"]` 식으로 슬래시까지 포함해 `Day: 1`이 `Day: 10`에 오매칭되는 것 방지 + 각 Day 고유 장소명 추가. ② 10일 일정을 1개 압축 constant 로어로 통합(단 상시 로어 4개가 되므로 description 일부 흡수로 상쇄). → F1·F3을 함께 고려하면 **D 디렉터 권고("일정을 분단위 강제 스케줄 → 상황 카드 풀로 격하")**와 결합하는 게 최선: 일정 로어를 "그 무렵 쓸 수 있는 장면 후보"로 축약 + 키를 장소명/스테이터스 표기 기반으로.

### F4. [H / A·E] 위 F1~F3 복합 결과 — 전개 일관성 붕괴
일정 로어 미트리거 → AI가 일정 임의 생성 → 작가 의도 타임라인과 어긋남. Day-Phase 매핑 무의미화. F1~F3 수정 시 동반 해소.

### F5. [컬처 / K] 방콕 지리·고유명사 오류군 🔴
- "방콕 외곽 풀빌라"인데 매일 시내 핵심부(왓아룬·아이콘시암·짜뚜짝·실롬) 왕복 — 지리적으로 비현실 (외곽↔시내 편도 1~1.5h)
- 로어 [1109] "Vertigo (방나타워 61층)" → **틀림**. Vertigo는 사톤 Banyan Tree Bangkok 61층. 방나(Bangna)는 동남부 다른 지역
- Day 6 "팟퐁 야시장" — 팟퐁은 성인업소 밀집 구역, 350만 인플루언서 콘텐츠 동선에 부적합
- Day 8 담넌사두악 수상시장 — 방콕에서 편도 1.5~2h, 왕복 이동시간 누락 → 그날 밤 디너크루즈와 시간 충돌
- **수정안**: 숙소 위치를 명확히(방콕 동부 특정 + 시내 이동시간 명시, 또는 근교 휴양지로) / Vertigo → "반얀트리 방콕 61층, 사톤" 정정 또는 일반화 / 팟퐁 → 아시아티크·조드페어 야시장으로 교체 / 담넌사두악 → 이동시간 반영 또는 암파와로 교체
