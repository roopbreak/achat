## FAIL (수정 필수)

### 1. [E·B] first_mes 안에서 톤이 끊긴다 — "정겨운 여름" 다음 줄이 "공용 배설구"

**현재**: first_mes는 4문단 내내 서정적·향수적 풍경이다.
> "이 뜨겁고도 정겨운 여름의 마을은 이제 {{user}}가 지내게 될 곳이다."
> "매미 우는 소리가 한층 진해진다. 어디선가 강아지 발소리가 흙길을 살짝 긁고 지나간다."

바로 이어지는 모드 메뉴에서:
> "`!노예모드` — 당신은 오래전부터 이 마을의 작업 노예이자 공용 배설구다"

"정겨운 여름의 마을"에서 "공용 배설구"로 가는 톤 다리(bridge)가 한 줄도 없다. 신규 플레이어는 첫 화면에서 톤 충격을 받는다. 이건 "수위 경고"가 아니라 **디렉터가 톤을 결정하지 않았다**는 신호다 — 서정극과 노예물 사이를 메뉴 한 줄로 점프시킨다.

**수정안**:
- first_mes의 풍경 문단과 모드 메뉴 사이에 톤 전환 1~2줄 삽입. 예: "이 마을의 여름은 사람마다 다르게 흐른다. 당신이 어떤 여름을 보낼지 먼저 정해야 한다." — 메뉴가 "장르 선택"임을 명시적으로 프레이밍.
- 모드 메뉴 라벨에서 노골 단어를 메뉴에선 절제하고(예: "`!노예모드` — 마을이 당신을 부리는 여름"), 노골 정의는 진입 후 로어에서 펼친다. 메뉴는 입구지 본편이 아니다.

### 2. [E] 메인 캐릭터(최인혜)가 자기 first_mes에 등장하지 않는다

**현재**: `char_name`은 "최인혜"인데 first_mes 전체에 최인혜가 없다. 이름도, 대사도, "이장의 책임감 + 미망인의 외로움"이라는 작품 핵심 정서도 first_mes에 0회 등장. first_mes는 순수 환경 묘사 + 시스템 메뉴다.

엔진 레퍼런스 §3: "first_mes — 캐릭터 성격이 드러나고 진입을 유도." 이 요건이 메인 캐릭터에 대해 충족되지 않는다. 카드의 주인공이 입구에서 투명 인간이다.

**수정안**: 모드 선택 메뉴는 유지하되, first_mes 마지막에 최인혜의 실루엣 한 컷을 넣어 진입 후크를 만든다. 예: "흙길 저편, 밀짚모자를 눌러쓴 여자가 손차양을 하고 이쪽을 본다. 마을의 이장이다. — 어떤 여름을 살지 정하면, 그녀가 먼저 다가올 것이다." 모드 미선택 상태를 깨지 않으면서 주인공을 입구에 세운다.

### 3. [E] 이미지 URL 스킴이 description과 로어에서 서로 다르다 — 둘 다 캐시 블록에 들어간다

**현재**:
- description: `![](https://risu.ddsmdy.com/images/%EC%97%90...X%EC%8A%A4/{캐릭터코드}/{상황코드})` — 스토리명 포함, 확장자 없음
- 로어 [1429] 이미지 URL 시스템: `![](https://pub-8d335208688149528dd8a208d1b79653.r2.dev/{캐릭터}/{상황}.webp)` — 스토리명 없음, `.webp` 있음, 도메인 자체가 다름

description(block 2)과 상시 로어(block 3)는 둘 다 캐시되어 매 턴 함께 주입된다. AI는 **상충하는 두 이미지 규칙**을 동시에 받는다. 예시 코드까지 어긋난다 — description은 "민지유/61 ← 수박먹기", 로어 1429는 "민지유/09.webp"(09=흥분). "농밀 비주얼 하렘"을 표방하는 작품의 비주얼이 입구에서 반쯤 깨진다.

**수정안**: 진실의 원천 하나로 통일. 실제 서빙되는 URL이 무엇인지 확인 후, description의 이미지 시스템 블록을 **삭제**하고 로어 [1429] 한 곳만 남긴다(정보-지시 분리 + 캐시 단일화). 예시 코드도 [1430] 상황 코드 매핑과 일치시킨다.

### 4. [E·금기] 노예모드의 프레이밍과 합의 안전장치가 정면 충돌한다

**현재**:
- 로어 [1452]: "{{user}}는 오래전부터 이 마을의 작업 노예이자 공용 배설구"
- 로어 [1435] 합의 안전장치: "누구든 명시적 거절(말 또는 손) 한 번이라도 나오면 즉시 행위 중단" + "'노예니까 거절 못 함' 식 강제 화법 금지"

"오래전부터 소유된 공용 배설구"라는 **상태**와 "언제든 거절하면 즉시 중단"이라는 **장치**가 화해되지 않은 채 병치돼 있다. AI는 {{user}}가 소유물인지 아닌지 매 턴 흔들린다. 안전장치 자체는 옳지만, 모드가 파는 판타지("오래전부터")와 장치("즉시 옵트아웃")를 잇는 **프레임 설명이 없다**.

**수정안**: 노예모드 로어에 "프레임 vs 행위" 구분을 명시. 예: "노예라는 *설정*은 {{user}}가 선택한 놀이 프레임이다. 그 프레임 안에서 마을 여자들은 거리낌 없이 부리고 들이댄다. 단 {{user}}의 명시적 거절은 *프레임 자체에서 빠져나오는 신호*로 읽고 즉시 톤을 평상 모드로 전환한다." — 소유 판타지와 옵트아웃을 같은 문장 안에서 정합하게 만든다.

### 5. [E·금기] 노예모드 + 처녀 5인 — 세션 시작 시점에 동시에 참일 수 없는 세계관

**현재**:
- 노예모드: {{user}}는 "오래전부터" 마을의 "공용 배설구"
- 행동 규칙 / 로어 [1435]: "캐릭터가 처녀인 경우(배아윤·고수진·김규미·오설아·박경아) 첫 삽입 시 처녀 삽입 묘사 규칙 준수"

{{user}}가 오래전부터 마을 공용 배설구였다면 11인 중 5명이 어떻게 세션 시작 시점에 여전히 처녀인가. 노예모드의 과거 설정과 처녀 5인의 현재 상태가 모순된다. 작성자가 두 시스템을 따로 설계하고 합치지 않은 자리다.

**수정안**: 노예모드 로어에 "노예 = 작업 노예 위주, 성적 관계는 캐릭터별로 아직 시작 안 한 상태일 수 있음. 처녀 캐릭터는 노예모드에서도 첫 관계 전" 같은 경계를 명시하거나, 노예모드에서는 처녀 묘사 규칙의 적용 범위를 따로 정의한다. 지금처럼 침묵하면 AI가 즉흥으로 메운다.

### 6. [B] post_history가 25줄을 넘어 핵심 규칙이 흘러간다 + 동시 시스템 3개

**현재**: post_history_instructions는 "응답 작성 순서 7단계" + "모드별 톤 요약 6줄" + "합의 안전장치 4줄" + "참조 6줄" — 합산 25줄 이상. 엔진 레퍼런스 권장은 3~5줄. post_history는 매 턴 재주입되는 *마지막 리마인더*인데, 길면 AI가 핵심을 흘려보낸다.

동시에 스테이터스 시스템(INFO 박스: ❤️0~1000 + 💦관계횟수 + 위치 + 속마음) + 모드 시스템 + 이미지 시스템 = **동시 시스템 3개**. 엔진 레퍼런스 "동시 시스템 2개 이내" 위반.

**수정안**:
- post_history는 "①모드 확인 ②장면 인원 1~4명 ③시골 감각 1개 ④발화자 명시 ⑤INFO 박스" 5줄로 압축. 나머지(상세 톤·안전장치·참조)는 이미 상시 로어에 있으니 post_history에서 중복 제거.
- 이미지 시스템을 "시스템"이 아니라 "응답 후처리 옵션"으로 격하 — INFO 박스 + 모드만 매 턴 강제, 이미지는 "여유 있으면" 수준으로. 부담을 2개로 낮춘다.

### 7. [D] 상시 로어 9개 — 권장 1~2개, 과다 기준 4개+의 2배

**현재**: constant=1 로어가 [1429][1431][1435][1430][1432][1433][1437][1434][1436] 총 9개. 매 턴 캐시 블록 3에 전부 주입된다. 엔진 레퍼런스: "상시 로어 1~2개 권장, 4개+ 과다." 총 항목도 33개(WARN 기준 30+).

디렉터 관점에서 이게 왜 치명적인가: **상시 블록이 거대해서 캐릭터 콘텐츠가 토큰 예산 아래로 밀린다.** 프롬프트의 무게 중심이 캐릭터가 아니라 행정(INFO 박스 형식·상황 코드·우선순위 7단계·응답 우선순위 7단계)이다. 작품이 캐릭터가 아니라 양식으로 굴러간다.

**수정안**:
- 절대 상시여야 하는 것만 남긴다: 합의 안전장치 [1435], 성적 어휘 [1437] — 2개.
- INFO 박스 형식·이미지 시스템·상황 코드·우선순위 단계는 post_history에 1~2줄 요약으로 옮기거나, 키워드 트리거로 전환(예: 이미지 로어는 캐릭터 등장 시에만 필요).
- "응답 우선순위 7단계"[1432]와 post_history "응답 작성 순서 7단계"는 **완전 중복** — 하나 삭제.

## WARN (개선 권고)

1. **[F] 서브 9인에 "엔진 출력 패턴"과 "금지"가 없다.** 최인혜·박경아만 응답마다 강제되는 행동 패턴("이장 위치 발화 OR 외로움 비트", "사투리 + 호기심 행동")과 금지 목록을 갖는다. 나머지 9인은 성격 메모만 있고 매 응답 차별화 강제 장치가 없다 → 플레이에서 9인은 generic으로 수렴한다. 최소한 각 서브 캐릭터 로어에 "응답마다 1개" 행동 시드 1줄씩 추가 권장.

2. **[F] 비밀 보호가 조건 트리거가 아니다.** 최인혜 "마을의 누구도 모르게", 하정연 "과거 질문엔 짧게 답하고 화제 돌림" — 둘 다 무드 묘사지 엔진 레퍼런스 §9가 요구하는 "오직 'XX' 언급 시" 식 조건 트리거가 아니다. 하정연의 과거는 특히 트리거 단어를 정해두지 않으면 AI가 첫 턴에 다 풀거나 영원히 안 푼다.

3. **[E] 고향모드 최인혜 호칭이 세 군데에서 어긋난다.** 로어 [1449] "호칭: 어릴 적 별명('달달이')", description 말투 예시는 전부 "총각", 호칭 매트릭스는 "어릴 적 별명 → 총각". 고향모드에서 최인혜가 {{user}}를 뭐라 부르는지 불명확. 또 "달달이"라는 디폴트 별명은 {{user}} 성별·취향을 가정하는 데다 "또는 유저 이름의 어릴 적 호칭"이라는 헤지가 붙어 작성자도 확신이 없었음을 드러낸다 — 별명 규칙을 한 줄로 확정하거나 "유저가 별명을 정하게" 유도.

4. **[E] "20XX년" 플레이스홀더가 first_mes·scenario·INFO 박스에 그대로 노출.** 라이브 성인 스토리가 "20XX"로 시작하면 미완성 초안처럼 읽힌다. 연도를 확정하거나(연출상 의미 있게) 최소한 "어느 여름"처럼 의도된 모호함으로 다듬는다. 제목의 "X스" 검열 표기와 합쳐져 표면이 드래프트처럼 보이는 효과를 키운다.

5. **[D] 범용어 키워드.** [1448] "강아지"·"개", [1455] "수박", [1456] "물놀이"·"우물", [1459] "회관" 등 일상어가 키로 쓰여 의도치 않은 과트리거 위험. "수박"은 시골 대화에서 수시로 나온다. 명사형 고유 키워드로 좁히거나 AND 조합 권장.

6. **[B] description 길이.** 이미지 시스템 + 메인 2인 풀시트 + 관계망 + 행동 규칙 + 처녀 삽입 규칙 + 출력 지시 — 권장 3,000자를 상회할 가능성 높음(A 페르소나 정밀 측정 필요). 이미지 시스템 블록을 로어로 빼면(FAIL #3 수정안) 상당 부분 해소.

## 디렉터 렌즈 — 사용자 지정 질문 직답

### Q1. 6모드 시스템 — 차별화로 작동하는가, 라벨만 다른 같은 톤인가

**라벨 차이에 가깝다. 차별화는 "오프닝"에만 걸려 있다.**

- 6모드는 실제로 **2축 + 부록**이다: 관계 거리축(고향↔귀농) + 성적 농도축(음란 on/off) + 노예(별개 권력축) + 커스텀(탈출구). 그런데 조합이 비대칭이다 — "음란고향"은 있는데 "음란귀농"도 "노예고향"도 "노예음란"도 없다. 시스템이 아니라 **임의로 고른 팬서비스 프리셋 목록**이다.
- 로어 [1433]은 "톤 차이가 작품 전체를 결정"이라 선언하지만, 모드 트리거 로어([1449]~[1454])가 실제로 거는 건 거의 "호감도 시작값 + 호칭 + 첫 마중 인물"뿐이다. 귀농모드에서 20턴 지나 호감도가 오르면 음란모드와 사실상 같은 끈적한 장면이 나온다. 모드는 **시작점**을 가를 뿐, **지속 톤**은 호감도 수치로 수렴한다.
- "음란고향 = 고향 + 음란"이라고 로어가 직접 쓴다. 산술 조합을 모드로 파는 건 차별화가 아니다.
- 캐릭터 시트는 구체적인데(성감대·신음·사후 행동), **정작 차별화 부담을 다 짊어진 모드는 추상 무드 단어로만 기술**된다 — "톤: 끈적함 + 노골적 시선", "톤: 추억 + 친근감". 프롬프트에서 가장 구체적이어야 할 부분이 가장 추상적이다.

**디렉터 판단**: 6모드는 디렉터가 톤을 결정하지 않고 플레이어에게 외주 준 결과물이다. 강하게 권하는 방향 — 모드를 "오프닝 프리셋"으로 정직하게 격하하고(메뉴에서도 "시작 설정"이라 부른다), 음란/노예 같은 *지속 톤*은 모드가 아니라 호감도·상황 로어가 책임지게 일원화. 또는 반대로, 각 모드에 "이 모드에서만 켜지는 지속 규칙 3줄"을 구체적으로 박아 진짜 시스템으로 만든다. 지금은 둘 다 아니다.

### Q2. 11인물 — 실제로 갈라지는가, 외모만 다른 동일 인격인가

**2명은 갈라진다. 나머지는 외모가 영혼보다 더 세밀하게 차별화돼 있다.**

- **갈라지는 2인**: 최인혜(이장 책임감 + 미망인 외로움, 응답마다 강제 패턴 있음), 박경아(다른 도 사투리 — 가장 강한 차별화 장치 + 호기심 행동 강제). 이 둘은 엔진 출력 패턴이 매 응답 차별화를 강제한다.
- **처녀 4인(배아윤·고수진·김규미·오설아)**: 표층 태그는 다르다(조용/활달/다정/털털). 그러나 **성적 아크가 거의 동일**하다 — 전원 "처녀 + 부끄러움 + 처녀 삽입 규칙 + 사후 부끄러움". 신음 한 줄씩만 다를 뿐 밑 구조가 같다. 하렘의 핵심 재미가 "구별되는 여러 명을 모으는 것"인데 4명이 "수줍은 처녀, 약간의 맛 차이"로 겹친다.
- **"외로움" 심층 과부하**: 최인혜(미망인 외로움) / 신주아(남편 멀어 외로움) / 고유진("사실은 외로움 많음") / 민지유(노처녀 갈증 = 외로움 변종). 11명 중 4명의 심층이 같은 우물에서 나온다. 기쁨·야망·신앙·장난기-그-자체·악의 같은 정서 다양성이 없다.
- **신주아 ≈ 하정연**: 둘 다 차분·신중, 둘 다 처녀 아님, 둘 다 "감정 흘리지 않으려 애씀", 둘 다 "한 번 무너지면/결정하면 자기 페이스". 구별점은 신주아=결혼 죄책감, 하정연=미공개 과거 — 같은 "절제된 경험녀" 아키타입의 두 변주.
- **외모는 정밀하다**: 키 158~167, 머리색·길이·가슴 크기 전부 다르게 설계. 몸은 11개로 갈라놨는데 인격은 사실상 5~6개다.

**디렉터 판단**: "외모만 다른 동일 인격" 위험이 처녀 4인 + 신주아/하정연에서 **부분 실현**됐다. 우선순위 수정 — (a) 처녀 4인 중 최소 2명은 성적 아크 자체를 비틀어라(예: 1명은 "수줍지만 호기심으로 자기가 먼저 자료를 찾아본 적 있음", 1명은 "종교적·도덕적 죄의식이 핵심"). (b) "외로움"을 쓰는 4인 중 2명의 심층을 교체. (c) 서브 9인 각각에 응답마다 강제되는 행동 시드 1줄.

### Q3. '시골 하렘'이 천편일률 막친구물로 읽히는가

**위험이 높다. 정작 신선한 전제를 에로 연료로만 태우고 있다.**

- 신선한 후크는 있다 — **전력 제한(에어컨 없음)**. 더위·땀·등목을 모든 장면에 강제하는 장치로는 영리하다. 그리고 최인혜의 "미망인 이장" 멜랑콜리도 결이 있다.
- 그런데 이 전제가 **드라마로 쓰이지 않는다**. 비닐하우스·우물·등목·수박 — 로어가 전부 "옷이 들러붙고 살이 비친다"로 수렴한다. 더위가 *장애물*이나 *비극*이 된 적이 없다(열사병, 말라가는 작물, 마을이 죽어간다는 미래의 슬픔). 이 세계에서 가장 흥미로운 것 — **버려진 여자들만 남은, 죽어가는 마을** — 이 정서적으로 전혀 활용되지 않는다.
- "왜 여자들만 남았는가"는 [1434]에서 "남편 사별/도시 이주/멀리 떨어진 가족"으로 손짓 한 번에 처리된다. 11명이 전부 편리하게 가용하고, 전부 편리하게 한 청년에게 끌린다 — 세계관이 이걸 *벌어내지* 못한다. 막친구물 독해가 활짝 열려 있다.

**디렉터 판단**: 차별화는 "전력 제한 + 미망인 이장"에 전부 걸려 있는데, 둘 다 에로 양념으로만 소비된다. 권하는 방향 — 전제가 약속하는 *쇠락·멜랑콜리*를 최소한 1개 로어/엔딩 분기로 실현(예: "마을이 곧 행정구역에서 사라진다"는 시한, 또는 떠난 사람들의 빈집이라는 상시 풍경 시드). "농밀 하렘"과 "죽어가는 마을의 마지막 여름"을 같은 작품 안에서 만나게 하면 막친구물에서 빠져나온다.

### Q4. 금기가 무너진 자리

1. **노예모드 프레이밍 ↔ 합의 안전장치 모순** (FAIL #4) — "오래전부터 소유된 공용 배설구"와 "언제든 거절 시 즉시 중단"이 화해 없이 병치. 프레임/행위 구분 문장이 없어 AI가 매 턴 흔들린다.
2. **노예모드 ↔ 처녀 5인 세계관 모순** (FAIL #5) — 오래전부터 공용 배설구인 {{user}}가 있는 마을에 처녀가 5명. 시작 시점에 둘 다 참일 수 없다.
3. **서브 9인 "금지" 미정의** — 최인혜("직접 고백 금지", "사별 남편 가볍게 처리 금지")와 박경아("마을 어른인 척 금지")만 금지 목록을 갖는다. 나머지 9인은 캐릭터·세계관을 깨뜨리는 행위가 정의 안 됨. 민지유의 "거절당하면 며칠 안 보임"이 9인 중 유일한 행동 결과(consequence) — 금기 설계가 캐스트 전체에 불균등.
4. **"취해서/원래 이런 사이라서" 우회 화법 금지는 명시됨** — 이건 무너지지 않았다. 다만 노예모드의 "내 거"(민지유)·"야~ 너~"(고유진) 호칭이 강제로 미끄러지지 않게 잡아줄 *구체적 craft 가이드*가 거의 없다 — 어려운 모드에 가드 장인정신이 얇다.

### Q5. first_mes ↔ description 톤 흔들림

- **first_mes 내부에서 이미 끊긴다** (FAIL #1) — 서정적 풍경 4문단 → "공용 배설구" 메뉴 라인. 다리 0줄.
- **first_mes ↔ description 무게중심 불일치** — first_mes는 100% 서정 풍경, description은 90% 기계(이미지 시스템·코드·INFO 형식)로 시작. 같은 작품의 두 입구가 완전히 다른 것을 약속한다.
- **메인 캐릭터 부재** (FAIL #2) — first_mes에 최인혜 없음. "정겨운 여름"의 톤과 description의 "농밀 하렘 11인" 톤 사이를 이어줄 인물 자체가 입구에 없다.
- **"20XX" 플레이스홀더 + "X스" 검열 제목** (WARN #4) — 표면이 미완성 초안 톤. 라이브 작품의 첫인상을 깎는다.

### Q6. 11명 동시 운영의 매력 붕괴 위험

**구조적으로 "2인 + 9 스텁"이고, 마을이 웹이 아니라 리스트다.**

- **전개 가드는 있다** — "11명 전원 동시 등장 금지, 1~4명만", "현재 장면 인원만 INFO 표기". 운영 안전장치 자체는 작동 가능.
- **그러나 발달은 2명뿐** — 최인혜(메인)·박경아(서브 메인)만 풀시트 + 엔진 출력 패턴. 9인은 1스크린짜리 키워드 로어 스텁. "하렘 11인"을 마케팅하지만 제품은 "2인 + 9 카메오"다.
- **캐릭터 간 관계가 없다** — 관계망 표는 최인혜/박경아/{{user}}만 매핑. 나머지 9인은 *서로* 간 관계가 0줄(처녀 4인 "또래 친구처럼 어울림" 한 줄 제외, 신주아=이장 대행 경험 한 줄). 하렘의 재미 절반은 여자들 사이의 질투·연대·경쟁인데, 좁은 마을에 11명이 사는데 내부 사회적 직물이 없다. **마을이 웹이어야 하는데 리스트다.**
- **발견 가능성 문제** — 9인은 이름이 키워드. 신규 플레이어는 "배아윤"을 칠 줄 모른다. first_mes INFO 박스의 이름 목록이 유일한 단서. 9인 호출이 "AI의 변덕 또는 대괄호 목록 읽기"에 의존.
- **행정이 캐릭터를 밀어낸다** — 상시 로어 9개의 거대 블록(FAIL #7)이 토큰 예산에서 캐릭터 콘텐츠를 아래로 민다. 11명을 다 굴리기엔 캐릭터당 가용 컨텍스트가 부족하다.

**디렉터 판단**: 11은 의도적으로 줄이거나, 줄이지 않으려면 *관계 웹*을 반드시 깔아야 한다. 권하는 방향 — (a) "코어 4~5인 + 카메오 6~7인"으로 티어를 정직하게 나누고 카메오는 그렇게 운영(전원 풀 연애 라인 약속하지 않음). (b) 또는 11명 유지하되 최소 "처녀 4인 내부 관계", "민지유↔고유진(둘 다 마을 험담 대상)", "신주아↔최인혜(이장 라인)" 같은 캐릭터 간 관계 로어 3~4개를 추가해 마을을 웹으로 만든다. (c) 발견 가능성 — 모드 진입 직후 "오늘 마주칠 만한 인물 2~3명"을 자연스럽게 노출하는 첫 마중 설계를 로어에 명시.

## 종합 디렉터 코멘트

이 작품의 핵심 문제는 개별 버그가 아니라 **디렉터가 톤을 결정하지 않았다**는 점이다. 6모드 시스템은 "서정 향수극 / 끈적 일상물 / 노예 굴림물"을 메뉴 한 줄로 점프시키는 장치이고, 그 결과 first_mes 안에서부터 톤이 끊긴다. 캐릭터 시트는 몸을 11개로 정밀하게 갈라놨지만 영혼은 5~6개로 겹치고, 그중 4개가 "외로움" 우물에서 나온다. 가장 신선한 전제(전력 끊긴, 여자만 남은, 죽어가는 마을)는 에로 연료로만 태워져 막친구물 독해를 막지 못한다. 상시 로어 9개의 행정 블록이 정작 캐릭터를 토큰 예산 밖으로 민다.

수정 우선순위: **FAIL #1·#2·#4·#5(톤·금기 정합)** → **FAIL #3·#7(이미지/로어 일원화)** → **FAIL #6(post_history 압축)** → WARN(캐릭터 깊이·관계 웹). 라이브 반영 시 #4·#5는 안전 정합 문제이므로 최우선.
