## FAIL 목록 (수정 필수)

### C/D군 — 엔진 작동 불능

**FAIL-1. [C·H·D] 모드 트리거 로어 6개 scan_depth=4 (본문 표기 "1"과 불일치)**
[1449]~[1454] 본문엔 `scan_depth: 1`이라 적혀 있으나 DB 실제값은 6개 전부 **4**. `lib/context-builder.mjs:257-267` `slice(-(depth*2))` → `!고향모드` 입력이 4턴간 스캔 윈도우 잔존, "첫 마중" 진입 지시가 4턴 반복 오염.
→ **수정**: 6개 엔트리 `scan_depth: 1`로 정정.

**FAIL-2. [D·H] 키워드 substring 충돌·오매칭**
`keywordMatch`는 순수 `includes()`(`lib/context-builder.mjs:230-252`). ① `"!음란고향모드".includes("고향모드")`=true → `!음란고향모드` 입력 시 [1453]+[1449] 동시 발화. ② [1448] 황구 keys에 `"개"`(1글자) → "전개/몇 개/개울/소개" 등에 매 턴 오매칭. ③ [1455~1461] 장소 로어 키가 "수박/우물/농장/회관/바다" 범용어 → 서술에 상시 등장해 사실상 상시화. ④ [1456]/[1461] 키 "우물" 중복.
→ **수정**: 모드 로어 부분 키("고향모드" 등 `!` 없는 형태) 제거하고 `!풀네임`만. 황구 `"개"` 삭제. 장소 로어 키를 구체 명사구로 좁히거나 통합(FAIL-7 참조).

**FAIL-3. [C] 이미지 시스템 — description·로어 두 경로 모두 죽음 (라이브 검증)**
description은 `risu.ddsmdy.com/images/{스토리}/{한글캐릭터명}/{코드}`, 상시 로어 [1429]는 `pub-...r2.dev/{한글캐릭터명}/{코드}.webp`. 둘 다 캐시 블록에 동시 주입.
**오케스트레이터 실측**: ① R2 외부 URL `최인혜/01.webp` → **404**. ② 내부 라우트는 `story_images.char_dir`와 매칭하는데 실제 char_dir은 **영문 2글자 코드**(BA·GY·HJ·IN·JK·KK·MD·MM·OS·PK·SM, 각 64장 scene_key 01~46/50~51/60~75 디스크+DB 실존) → 한글 캐릭터명 경로는 char_dir 불일치로 죽음. ③ `risu.ddsmdy.com/images/{스토리}/IN/01` (영문코드·확장자 없음) = **200 OK**. 즉 현 라이브 스토리의 이미지는 어느 경로로도 100% 작동 불능.
→ **수정 (Codex #3 채택, "옵션 A'")**: description의 `![](http..)` 리터럴 + 상시 로어 [1429] 외부 URL 지시를 **둘 다 제거** → `hasImageMapping=false` → 엔진 `buildImageSection`이 `story_images`의 실제 코드 기반 카탈로그를 자동 주입(`lib/context-builder.mjs:171,360-362`). scene_key 의미 매핑(01=기본…28=정상위…)은 [1430]을 "scene_key 의미표"로 정리해 1개 로어로 유지. **한글명→영문코드 추정 매핑은 라이브 반영에 부적절하므로 하드코딩하지 않음.**
