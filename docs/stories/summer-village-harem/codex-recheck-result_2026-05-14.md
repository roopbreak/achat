→ 액션: [1432]를 다시 쪼개라. 상시에는 “자동 카탈로그만 사용” 정도만 남기고, H/임신/웨딩/후반 체위 매핑은 모드·호감도·분기 키워드 로어로 내려야 한다.

3. **FAIL:** 모드 유지가 사실상 엔진 상태가 아니라 모델 기억에 의존한다. [1433]은 “현재 선택된 모드를 매 응답 유지”라고 쓰지만 DB/컨텍스트 빌더 어디에도 모드 상태를 저장하지 않는다. 게다가 INFO 줄은 `!고향` 같은 축약형이라 실제 트리거 키와도 안 맞아, 몇 턴 지나면 멀티캐릭터 장면에서 모드 드리프트가 날 가능성이 높다.  
→ 액션: 모드를 별도 상태값으로 저장하거나, 최소한 assistant가 매턴 동일한 exact token(`!고향모드` 등)을 상태창에 보존하게 바꿔라. 지금 구조는 “유지 지시”만 있고 “유지 장치”가 없다.

4. 보류 판단도 안일했다. HOLD-2의 질투·소문 시스템 부재는 “품질 개선”이 아니라 멀티캐릭터 상태 동기화의 핵심 결손이다. 현재는 [1431]이 현재 장면 1~4명만 표시하게 하고, 관계는 캐릭터별 `❤️/💞`만 따로 굴린다. 즉 여러 여성과 병행 관계가 벌어져도 다른 인물 반응·전파·충돌을 묶는 공용 규칙이 없다.  
→ 액션: 최소형 상시 규칙 1개라도 넣어라. “같은 날/같은 장소에서 목격·소문 나면 다음 응답부터 거리감/견제 반영” 정도의 가벼운 동기화 앵커가 없으면 멀티캐릭터물이 아니라 병렬 탭 모음이 된다.

5. NSFW 안전장치 자체는 남아 있지만, 1차 통합 과정에서 장르 가속 페달이 너무 앞당겨졌다. [1437]의 노골적 성적 어휘, [1432]의 H `scene_key` 카탈로그, `scenario`와 `first_mes`의 `!음란모드`/`!노예모드` 노출이 모두 P0부터 캐시에 깔린다. “모드 미선택 시 NSFW 금지”보다 “언제든 포르노로 꺾어라” 신호가 더 두껍다.  
→ 액션: 합의/거절/금지 블록만 상시로 남기고, 성적 어휘 가이드와 H 이미지 분류는 음란 계열 모드 또는 관계 단계 이후에만 들어오게 내려라. 지금은 안전장치보다 자극 지시가 먼저 보인다.

6. `first_mes`와 상태창은 아직도 트리거 오염원이다. `first_mes`는 {{user}} 행동을 노골적으로 선점하진 않지만, “마을 입구 도착”과 “최인혜가 먼저 걸어올 준비”를 이미 깔아 버려 멀티 오프닝을 사실상 단일 루트로 편향시킨다. 여기에 엔진 기본 `━━━` 스테이터스까지 병행 출력되면 assistant 메시지에 이름·장소·복장 정보가 누적되어 이후 키워드 로어를 더럽힌다.  
→ 액션: `first_mes`는 모드 선택 인터페이스만 남기고, 첫 마중 인물/장소는 모드 트리거 로어에서만 결정하게 재분리해라. 그리고 기본 `━━━` 상태창을 끄거나 INFO와 완전히 합쳐서 중복 상태 출력부터 막아라.
