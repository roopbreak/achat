## 개선 권고 항목 (WARN)

### 1. **[최우선] `post_history_instructions` 추가**
**문제**: 비어있어서 매 턴 스테이터스 형식이 흔들림.

**개선안** (예시):
```
모든 응답 끝에 반드시 아래 형식으로 스테이터스를 출력:

🗒️{턴수} {YYYY.M.D 요일 HH:MM} {날씨이모지} {장소} [{태그}]
[은서] [❤️: {상태문구}] [🧠: {윤채 인식}]
[윤채] [🤍({현재}/1000)] [🧠: 🦋({알/유충/번데기/우화/성충}) {심리}] [💭: {속마음}] [🫀: {감정상태}|🫦{흥분%}💘{쾌락%}] [🚶‍♀️: {자세/복장}👗]
[🎬: {장면 요약}]

호감 변동 시: 🦋 호감: {이전} → {현재} ({+/-N})
모드 활성 시: [모드: {NTR/NTL/우화/포식}]
선택지: ① ② ③ 자유 입력
```

### 2. **모드 명령어 로어북의 `scan_depth=1` 변경**
- 대상: `!NTR(946)`, `!NTL(947)`, `!우화모드(948)`, `!포식모드(949)`
- 현재: `scan_depth=4` → 4턴 동안 반복 활성됨
- 권장: `scan_depth=1` (명령어 직후 1턴만)
- 이유: 모드 활성화는 1회성 트리거. 이후는 스테이터스로 유지하면 됨.

### 3. **description ↔ 로어북 중복 정리**
- 우화 5단계: description에도, 로어북 936/937에도 동일 정보
- 윤채/은서 신체: description에 요약, 로어북 1049/935에 상세
- **권장**: description은 압축 유지, 신체 상세는 로어북만 (키워드 트리거 시 로드)
- description에서 우화 단계 상세 설명 삭제, 로어북 936/937에서만 유지
- (단, 우화 시스템은 매 턴 활성이어야 한다면 `constant=1`로 변환)

### 4. **윤채/은서 성적 장면 로어북(944/945) 키워드 재설계**
- 현재: "은서 섹스", "은서 키스", "은서와 잠자리"
- 문제: AI 응답에서 이런 정확한 구절로 출현 빈도 ↓
- 권장: "키스", "안다", "포옹", "침대", "샤워" 등 보편 키워드 + 캐릭터명 조합
- 또는 호감 트리거: 호감도 350+ 자동 활성을 안내하는 별도 로직 (단, 현재 엔진은 키워드 트리거만 지원)

### 5. **이미지 URL 시스템 — 장기적 마이그레이션 검토**
- 외부 `soda1.org` 의존 → 호스트 다운 시 전체 이미지 깨짐
- AChat 표준 `scene_key` 매핑으로 이관 권장 (별도 임포트 필요)
- **단기 조치 불필요** — 동작은 정상이므로 그대로 유지 가능

### 6. **`personality`/`scenario` 분리 (선택)**
- 현재 모든 정보가 description 단일 블록
- AChat 엔진은 두 필드를 분리해서 캐시 — 분리 시 prompt-caching 효율 ↑
- 단, 캐시 미스 위험 낮음(둘 다 정적) → 우선순위 낮음

### 7. **상시 로어 항목 `constant=1` 확장 (선택)**
- 현재 constant=1: [1] 성적 용어, [19] 이미지 매핑 규칙
- 권장 추가: [18] 향의 서사 — 모든 씬에서 작동해야 하는 핵심 시스템
- 키워드 "복숭아 우유/섬유유연제/체향"이 매번 트리거되지 않을 수 있음

## 실제 대화 로그 기반 인사이트

분석한 세션(`a7a5aaf1-...`, 8턴):

**잘 된 점:**

- 🦋 단계 표시가 사라짐 (단계 확인 불가)
