## 맥락: 1차 수정 완료 후 표본 재검수
이 스토리는 페르소나 6인 검수 → FAIL 풀-리라이트 → 라이브 DB 반영까지 끝난 상태다. 1차 사이클의 페르소나 검수와 Codex 검수가 모두 **서브에이전트 컨텍스트**에서 수행됐는데, 같은 하네스의 다른 스토리("너 쌓여있잖아")를 메인 세션에서 재검수했더니 서브에이전트가 놓친 신규 FAIL 2건이 나왔다. 그래서 이 스토리도 메인 세션에서 표본 재검수하는 것이다. 특히 이 스토리는 **멀티 캐릭터(간호사단)** — 캐릭터 간 충돌·전환·동기화는 서브에이전트 인라인 검수가 가장 놓치기 쉬운 영역이다. 너는 "이미 1차 수정된 라이브 스토리"를 본다 — 페르소나/Codex가 놓친 잔존 결함과, 적용된 수정안 자체의 부작용·회귀 위험을 잡아야 한다.

## 엔진 핵심 제약 (코드로 확증됨 — lib/context-builder.mjs, lib/db.mjs)
- keywordMatch는 순수 substring includes() — 키워드가 입력/출력 텍스트에 그대로 등장해야 매칭
- matchLoreByKeyword는 recentMsgs.slice(-(scan_depth*2)) + userInput을 스캔
- constant=1 로어는 매 턴 캐시 블록에 전부 주입, insertion_order로만 정렬(priority 정렬 아님)
- hasImageMapping = description에 마크다운 이미지 URL 있으면 자동 이미지 시스템 스킵
- AI는 턴/날짜를 못 셈. 동시 시스템 2개 이내 권장. 상시 로어 1~2개 권장(4개+ 과다)
