## 이미지 (2,080장 확보 — v1 범위표 방식, 자동 카탈로그 비활성)
- 실존 매트릭스: 캐릭터 27코드×~71장(상황 1~61, 80~82, 100~106) + c1 해밀턴 13장(일상 위주+104) + ts1 70장 + uu 72장(110·111 전용) + 장소 gr 10장 = **2,080장**
- b6 세니카는 106(샤워) 없음 / ts1·c3 르웬은 103(이능 사용) 없음 / uu는 103 의도적 제외
- **scene_key = `{캐릭터코드}-{상황코드}`** (예: `a1-20`, `gr-3`) flat 업로드
- **자동 카탈로그(2,080행 나열)는 토큰 폭발 → 비활성 필수**: description에 절대 URL 형식 예시(`![](https://risu.ddsmdy.com/images/villain-bureau/a1-1)`)를 포함하면 hasImageMapping 정규식(`!\[.*?\]\(https?://`)이 매칭되어 자동 카탈로그가 꺼진다 (v1 gf-phone 검증된 방식, lib/context-builder.mjs L556 부근)
- description에 **컴팩트 코드 범위표**(캐릭터코드 28줄 + 상황코드 카테고리 ~10줄)를 넣어 AI가 코드를 조합하게 한다 — 원작도 동일 방식으로 동작했음
- 출력 형식은 절대 URL(`https://risu.ddsmdy.com/images/...`) — 상대 URL 예시를 쓰면 카탈로그 비활성이 안 됨
