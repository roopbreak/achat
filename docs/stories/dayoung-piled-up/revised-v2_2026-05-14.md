## CR5 (안전 — 최우선) — "가까운 친척" 제거 + 중단 신호 재정의

### 문제
- `[1289]` 호칭 매트릭스: `최수영 → {{user}} = "아저씨"(기본) 또는 "오빠"(가까운 친척·동네 형 컨셉)` — **"가까운 친척" 허용 문구**. description 수영 「엔진 출력 패턴」·「관계망」 표에도 동일 문구. 18세 처녀 + 교복 + 처녀 삽입 + 권력 비대칭 위에 **근친 인접 신호**가 겹침.
- 합의 안전장치가 `[1286]` 한 덩어리 안에 묻혀 있어 수영의 두려움 발화가 중단 신호로 안 읽힐 여지.

### 2차 수정
1. **`[1289]` / description 전부에서 "가까운 친척" 문구 즉시 삭제**. 수영의 {{user}} 호칭은 **"아저씨"(남편 친구·후배 — 기본)** 또는 **"오빠"(동네 형·엄마 지인 — 비가족 외부인)** 두 가지만. 가족·친척 컨텍스트 완전 차단.
2. **수영의 "엄마/아빠 들킬까" 공포 발화를 1회 이하로 축소** — 기존 "1~2회"를 "최대 1회"로. 공포가 자극 장치로 반복되지 않게.
3. **합의 안전장치를 `[1286]`에서 독립**시켜 별도 상시 로어 `[1289]` 호칭·어휘 로어 안의 독립 블록 또는 `[1286]` 최상단으로 끌어올림 → 2차안: **`[1286]`을 분리(CR1)하면서 합의 안전장치를 `[1289]` 상단 독립 블록으로 이관** (P1부터 항상 보이는 상시 로어, 운영 수치와 분리).
4. **명시 거절·경직·회피를 중단 신호로 강하게 재정의** — "명시적 거절(말)" 뿐 아니라 **몸이 굳음·시선을 피하고 대답을 안 함·뒤로 물러섬·"잠깐만요" 등 망설임**도 즉시 중단 신호. 합의를 의심도/14cm 같은 수치 시스템에 묶지 말고 독립적으로 최상위 규칙으로 명시.
