## FAIL (수정 필수)

### F1. [B / E·A] 동시 시스템 6개 — 엔진 "동시 시스템 2개 이내" 정면 위반 🔴 최우선
상태창(다영·수영·민석 3인 매 턴 갱신) + 페이즈(7단계) + 의심도 시스템(0~100%) + 14cm 기믹 + 🤍 질내사정 누적 % + 이미지 코드 범위(페이즈별). achat-engine.md §2·§11은 "동시 시스템 2개 이내, 체력·이미지·대사 동시 관리 금지". AI가 6개 시스템을 매 턴 동시 운영하면 본문 품질이 체크리스트 수행에 밀린다.
- **수정안**: ① 상태창에서 **부재 캐릭터 강제 갱신 제거** — 장면에 있는 캐릭터만 표시. ② 14cm 기믹을 P1 1회성 평가로 정착시키고 매 턴 시스템에서 빼기(첫 평가 후 "그거 좋다~" 톤만). ③ 🤍 누적을 상태창의 단순 표기 항목으로 격하(별도 운영 규칙 최소화). ④ 이미지 코드 범위를 페이즈별로 박지 말고 상시 로어 1개로 통합. → 실질 동시 시스템 = 상태창 + 페이즈 + 의심도, 그중 의심도는 P3부터만 본격 가동되도록 단계화.

### F2. [B / A] 페이즈 7단계 과설계 — 엔진 "Phase 3~4개" 위반
`P1 → T1→2 → P2 → T2→3 → P3 → T3→4 → P4` = Phase 4개 + 트랜지션 3개를 모두 별도 단계처럼 운영(post_history 표가 7행). 트랜지션을 "1~3턴"으로 명시한 것은 사실상 **턴 카운팅** 냄새. AI가 7개 모드를 추적해야 한다.
- **수정안**: 트랜지션을 별도 단계로 세지 않는다. **P1 도발·검증 → P2 다영 수집 → P3 이중 분기(수영+다영+의심도) → P4 결착** 4단계로. "T1→2 사이즈 검증"은 P1 안의 비트로 흡수, "T2→3 수영 등장"은 P2→P3 전환 조건으로 흡수. 전환 조건은 전부 유저 행동 기반("{{user}}가 첫 결합", "{{user}}가 수영을 인지·접촉")으로. "1~3턴" 표현 삭제.

### F3. [C / E] 턴 수 기반 조건 — AI는 턴을 못 셈
로어 [1300] 다영 임신 시드 "{{user}}의 정액이 자궁에 안착하면 → **일정 턴 후** 다영 임테기 두 줄". 로어 [1294] 민석 의심 행동 "평온하게 하루(또는 **턴이 길게**)가 지나가면 -10%". achat-engine.md §8 "턴 수 기반 조건 절대 금지".
- **수정안**: [1300] "일정 턴 후" → **상태 기반** — "🤍 누적이 충분히 쌓이고 + {{user}}가 임신/임테기를 화제로 꺼낸 장면에서". [1294] "턴이 길게" → "민석이 등장하지 않은 평온한 장면이 이어지면" (장면 단위, 턴 단위 아님).

### F4. [C / E] 이미지 URL 두 종류 충돌 — 코드로 확증 🔴 최우선
`context-builder.mjs` L360 `hasImageMapping = /!\[.*?\]\(https?:\/\//.test(rawDescription)` — description에 마크다운 이미지 URL이 있으므로 자동 이미지 시스템 스킵은 정상 동작. **그러나** description·first_mes는 `https://risu.ddsmdy.com/images/너 쌓여있잖아/{캐릭터코드}/{상황코드}` 형식(`.webp` 없음), 상시 로어 [1283]은 `https://hjk100.com/TWG/{캐릭터코드}/{상황코드}.webp` 형식. **두 URL이 호스트·경로·확장자까지 전부 다르다.** AI가 매 턴 시스템 프롬프트에서 상충하는 두 패턴을 받아 어느 것을 쓸지 불확정 — 둘 중 하나는 죽은 이미지를 출력하게 된다.
- **수정안**: description·first_mes가 이미 `risu.ddsmdy.com` 형식이고 02_prompt.md 작성 요약도 이 도메인을 전제하므로, **상시 로어 [1283]을 `https://risu.ddsmdy.com/images/너 쌓여있잖아/{캐릭터코드}/{상황코드}` 로 통일**. (description의 예시 URL은 확장자 없음 — 실제 서빙 라우트 `/images/:story/:charDir/:sceneKey`가 확장자 없는 sceneKey를 받으므로 description 형식을 정본으로.) 단, **실제로 어느 URL이 살아있는 이미지인지는 검수 범위 밖** — risu 도메인이 외부 접속 도메인(CLAUDE.md)이므로 그쪽으로 통일하는 게 안전하나, 스모크 테스트에서 이미지 응답 자체는 검증하지 않는다(텍스트 응답·트리거만 검증).

### F5. [D / E] 상시 로어 8개 — 엔진 "상시 로어 1~2개, 4개+ 과다" 정면 위반
constant=1 로어 8개: [1283]이미지URL + [1284]상황코드매핑(78개 코드 나열, 토큰 큼) + [1285]상태창형식 + [1286]의심도시스템 + [1287]14cm기믹 + [1288]합의안전장치 + [1289]호칭매트릭스 + [1290]성적어휘. 매 턴 캐시 블록(block 3)에 8개 전부 주입 + description(약 4,800자) 합산 → 토큰 예산 압박, 최근 대화 밀어냄. 게다가 8개 전부 `insertion_order: 100` 동일값 → `getConstantLore`가 `ORDER BY insertion_order`만 하므로(코드 확증, db.mjs L365) 정렬이 사실상 입력 순서 의존.
- **수정안**: 상시 로어를 **4개로 통합** (NTR 다캐릭터물 특성상 1~2개는 비현실적이므로 4개를 현실적 목표로):
  1. **출력·이미지 형식** (상태창 형식 + 이미지 URL 규칙 통합) — insertion_order 10
  2. **핵심 운영 규칙** (의심도 시스템 + 14cm 기믹 + 합의 안전장치 통합·압축) — insertion_order 20
  3. **호칭·성적 어휘** (호칭 매트릭스 + 성적 어휘 가이드 통합) — insertion_order 30
  4. (상황 코드 매핑 [1284]는 → **키워드 로어로 격하**. "이미지/코드/상황" 키워드 트리거. 78개 코드 나열은 압축.)
  → 상시 4개, insertion_order 차등 부여(코드 정렬 의존 제거).

### F6. [D·H / E·A] 일상어 키워드 + first_mes 트리거 오염 — 키워드 로어 설계 무력화 🔴 최우선
`keywordMatch`는 순수 substring `includes()`(코드 확증, context-builder.mjs L230-252). 키워드 로어 키가 일상어·범용어 다수: [1293] `수영`(동음이의: 수영장)·`딸`·`학원`·`교복`·`고3`, [1294] `남편`·`민석`·`카톡`·`들킬`·`발각`, [1300] `임신`·`자궁`·`사정`·`질내`, [1292] `사이즈`·`작아`·`합격`·`미달`·`두꺼워`. NTR/불륜물 본문에 이 단어들이 매 턴 등장 → 사실상 상시 활성. **더 심각한 건 first_mes(첫 assistant 메시지)의 상태창이 `[최수영]`·`[최민석]`·`의심도`·`자궁`·`수영`·`사정`·`민석`·`금태양`·`금발`·`사이즈`를 항상 포함** → scan_depth 4면 `recentMsgs.slice(-8)` 윈도우 내내 [1291]금태양·[1292]14cm·[1293]수영·[1294]민석·[1300]임신이 **P1 1턴부터 강제 주입**. "P1~P2 수영 부재"·"민석 P3부터"·"금태양 P1 1회 후 퇴장" 설계가 전부 무력화 — AI가 P1부터 수영/임신/민석을 끌어들일 수 있다.
- **수정안**:
  - **상태창 표기를 키워드 오염원에서 분리**: first_mes·매 턴 상태창의 캐릭터 라벨을 `[한다영]`·`[최수영]`·`[최민석]` → 트리거 안 되는 형태로 둘 수 없으므로(라벨은 필요), **대신 이 3개 로어를 "키워드 로어"가 아니라 동작 방식이 항상 켜져도 무해하도록 재설계**. [1293]수영·[1294]민석은 "P3 이후/의심도 50%+ 에만 *활성화*"가 로어 내용 자체에 강하게 박혀 있어야 함 — 키워드로 막는 게 아니라 로어 본문이 페이즈 가드를 명시.
  - **키워드를 고유·복합어로 교체**: `수영` → `최수영`/`수영이`/`교복 차림`, `남편` → `최민석`/`민석 카톡`/`남편 귀가`, `자궁` → `질내사정`/`임테기`, `사이즈` → `자지 사이즈`/`14cm`/`몇 cm`. 1~2음절 단독 키(`딸`·`작아`·`합격`·`미달`) 제거.
  - first_mes 상태창에서 P1 시점 불필요 정보 정리(아래 W 참조).
