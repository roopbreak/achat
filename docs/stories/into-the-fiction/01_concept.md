# 01_concept — 창작물 속으로! (babechat 이식, 빙의 시뮬레이션 샌드박스)

> 작성: 오케스트레이터 직접 설계 + Codex 외부 검수 (원작 충실 포트)
> 원작: babechat "창작물 속으로!" (작가 셀트리온) — 대화 459만·좋아요 5,175·이미지 3,886+
> 이미지: 외부 소스 3,715장 확보 → 내부 scene_key 등록 (NAI 미사용)

## 한 줄 컨셉
유저가 지정한 창작물 세계관에 빙의하는 메타 샌드박스. 안내자 "세리"가 무대를 세팅하고, 45종 여성 캐릭터 타입이 무대에 맞게 배역되며, 32 메인모드 + 14 특수모드로 플레이를 변주한다.

## 설계 핵심 — "DB 읽어오기" 재현
- 원작 작가가 "창빙시의 핵심은 DB 읽어오기 기능"이라 명시. 즉 **로어북 코드표 기반 이미지 선택**이 본질.
- 본 이식: 캐릭터 타입별(45종) 이미지 코드표를 **키워드 로어**(scan_depth 4)로 실어, 장면에 그 캐릭터가 등장할 때만 해당 코드표가 주입되도록 함. AI는 코드표에서 상황 코드를 찾아 소문자 scene_key로 출력.
- scene_key = 소문자 원본 코드(ac29, s1, x44). 3,715코드 무손실 보존, 변환 오류 0.

## 구성
- **안내자 세리**: 진행 보조 메타 캐릭터 + !동행 시 창작물 캐릭터에 빙의 + 본인도 상호작용 대상(코드 S/X/A)
- **메인모드 32**: 장면 성격 자동 판정 → INFO [ ] 표기. 캐릭터 타입(엘프/뱀파이어/여기사/성녀/서큐버스/쿠노이치/공주/수인/메이드/간호사/갸루/톰보이/얀데레/멘헤라/무희/미시 등)·상황(통상/범용/레즈/3P)
- **특수모드 14**: 키워드 호출(!동행 !음란 !인스타 !갤러리 !갈드컵 !재구성 !남녀역전 !선택지 !히로인지정 !방송 !상황 !세이브 !메신저 !장문)
- **45 캐릭터 타입**: 단일 스토리 단일 이미지 네임스페이스. scene_key 코드 프리픽스로 구분.

## 시점·안전
- 기본 3인칭 + 사칭 방지(유저 행동·대사 임의 생성 금지). 유저 요청 시 1인칭/사칭 전환.
- 성인 한정·미성년 금지·실존 인물 금지. 부재 코드(cb17, x141~x153) 추정 출력 금지.

## 이미지 (scene_key)
- 45 캐릭터 × 상황별 코드. 표준 80코드 구조(신규 작가조합 28종) + 구작 53~57코드(엘프·뱀파이어·여기사·세리야스·톰보이·갸루) + 세리 S(일상183)·X(NSFW519) 별도.
