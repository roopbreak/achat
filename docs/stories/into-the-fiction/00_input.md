# 00_input — 창작물 속으로! (babechat 이식, 빙의 시뮬레이션 샌드박스)

## 사용자 원본 입력 (babechat 원작 수집)

소설·영화·게임·웹툰·만화·드라마 등 원하는 세계관 속 캐릭터로 빙의하는 시뮬레이션. 안내자 "세리"가 유저의 창작물 여행을 돕는다. 메인모드 32, 특수모드 14. 캐릭터 타입 45종(세리/엘프/뱀파이어/여기사/성녀/서큐버스/쿠노이치/공주/수인/메이드/간호사/노예 등) 각각 이미지 에셋 보유.

## 분석

### 장르
- 메타 샌드박스 / 빙의 시뮬레이션 + 성인 (판타지·현대·애니 무대 무관)
- 분위기: 유저가 무대(창작물 세계관)를 지정하면 그 안의 캐릭터로 진입, 안내자 세리가 진행 보조

### 구성 모드
- **멀티 캐릭터(45종) 단일 스토리**: import/images charDir='' 단일 네임스페이스. scene_key 프리픽스(코드)로 캐릭터 구분.
- 세리 = 안내자 + 동행(빙의) 가능한 핵심 캐릭터

### 핵심 시스템
- **메인모드 32**: 상태창 INFO 옆 [ ]에 현재 상황 자동 표기(💬통상/🔞범용/✂️레즈/3️⃣3P/🧝‍♀️엘프/🧛‍♀️뱀파이어/♞여기사/💋서큐버스/⚜️성녀/🥷쿠노이치 등)
- **특수모드 14**: 키워드 호출(!동행/!음란/!인스타/!갤러리/!갈드컵/!재구성/!남녀역전/!선택지/!히로인지정/!방송/!상황/!세이브/!메신저/!장문)
- **이미지 "DB"**: 원작은 로어북 코드표로 캐릭터별 상황→이미지코드를 AI가 조회해 출력(작가가 "핵심은 DB 읽어오기 기능"이라 명시). 본 이식도 동일 — 키워드 트리거 로어북에 캐릭터별 코드표를 실어 재현.

### 옵션
- 이미지: 외부 소스(serilll.org/IN/{code}.webp) 확보됨. NAI 생성 불필요. 내부 scene_key=소문자 원본코드로 등록(3,715장).
- 상태창: 원작 INFO + 모드 표기 + 동행칸 계승.

## 이식 설계 결정
- **scene_key = 소문자 원본 코드**(ac29, s1, x44 …): 3,715코드·45캐릭터 규모에서 의미 슬러그 하드매핑은 변환 오류 위험이 큼. 원작 선택 메커니즘이 "로어북 코드표"이므로, 코드를 무손실 보존하고 로어북 코드표를 그대로 실어 원작 방식을 충실 재현(변환 오류 0). 의미는 로어북 캐릭터별 코드표가 담당.
