AChat 인터랙티브 픽션 스토리 "모모"(캐릭터: 모모)를 신랄하게 재검수해주세요. "좋네요"는 빼고 약점만 날카롭게.

## 맥락: 1차 수정 완료 후 표본 재검수
이 스토리는 페르소나 6인 검수 → FAIL 풀-리라이트 → 라이브 DB 반영까지 끝난 상태다. 1차 사이클의 페르소나 검수와 Codex 검수가 모두 **서브에이전트 컨텍스트**에서 수행됐는데, 같은 하네스의 다른 스토리("너 쌓여있잖아")를 메인 세션에서 재검수했더니 서브에이전트가 놓친 신규 FAIL 2건이 나왔다. 그래서 이 스토리도 메인 세션에서 표본 재검수하는 것이다. 너는 "이미 1차 수정된 라이브 스토리"를 본다 — 페르소나/Codex가 놓친 잔존 결함과, 적용된 수정안 자체의 부작용·회귀 위험을 잡아야 한다.

## 엔진 핵심 제약 (코드로 확증됨 — lib/context-builder.mjs, lib/db.mjs)
- keywordMatch는 순수 substring includes() — 키워드가 입력/출력 텍스트에 그대로 등장해야 매칭
- matchLoreByKeyword는 recentMsgs.slice(-(scan_depth*2)) + userInput을 스캔
- constant=1 로어는 매 턴 캐시 블록에 전부 주입, insertion_order로만 정렬(priority 정렬 아님)
- hasImageMapping = description에 마크다운 이미지 URL 있으면 자동 이미지 시스템 스킵
- AI는 턴/날짜를 못 셈. 동시 시스템 2개 이내 권장. 상시 로어 1~2개 권장(4개+ 과다)

## 스토리 개요
"모모" — NTR/NTL 이중 루트 구조의 성인 인터랙티브 픽션. 1차 풀-리라이트에서 description 5867→3153자 감축, 상시 로어 9→7개, 이미지 URL을 risu.ddsmdy.com 형식으로 통일, [1307] 펨돔 모드 삭제, 타락도 시스템 통합. 1차 Codex 검수는 11/12 채택(C7 = 상황 코드 248 전수표 상시 제거만 보류).

## 검수 항목 (각 1~3줄, 액션 아이템은 "→ 액션:" 형식)
1. 페르소나 6인 + 오케스트레이터가 놓친 신규 엔진 위반·구조 결함이 있는가
2. 적용된 수정안의 라이브 부작용·회귀 위험 — 특히 상시 로어 통합 후 정보 누락/규칙 간 간섭, 페이즈·루트 전환 조건 모호성, 키워드 교체 후에도 남은 범용 키
3. 상시 로어 7개가 여전히 엔진 권장(1~2개) 대비 과다인지, 미래 정보(후반 루트·타락도 후반 단계 등)가 P1부터 캐시로 새는지
4. 우선순위 오판 — 03_qa_report.md에서 보류한 항목(C7 상황 코드 전수표 등)이 정말 보류해도 되는지
5. NTR/NTL 이중 루트의 안전성 — 합의 안전장치가 통합 과정에서 약해지지 않았는지
6. first_mes·상태창이 키워드 로어 트리거 오염원인지
