## 실행 메모 (서브에이전트 환경 제약)

이 검수는 서브에이전트로 실행되어 `Task`/`Agent` 도구로 페르소나 6인을 병렬 spawn할 수 없다. 따라서 `.claude/agents/persona-{director,chat-writer,engine-qa,nsfw-writer,psychologist,culture}.md` 6개 정의 파일을 직접 Read하고 각 페르소나 렌즈를 **인라인으로 순차 집행**했다. 엔진 위반 의심은 추측하지 않고 `lib/context-builder.mjs`(keywordMatch substring 매칭, post_history 재주입 슬롯, constant 로어 insertion_order 정렬)를 코드로 직접 읽어 확증했다. ## 종합 판정: **FAIL**

FAIL 3건 (E·A·N), WARN 다수. 핵심 원인 2가지:
1. **`post_history_instructions`가 빈 문자열** — 멀티 캐릭터 듀얼 스테이터스 + 3종 호감도 스탯 + 🔞경험 스탯 + 선택지라는 복잡한 매 턴 출력 시스템의 유일한 재주입 슬롯이 비어 있다. first_mes 예시만으로 1~3턴 안에 포맷이 붕괴한다.
2. **비밀이 비밀이 아님** (Codex #9 — 페르소나가 놓침) — 하은의 과거 편력·서연의 스토킹이 `description`(캐시 블록, 매 턴 상시 노출)에 박혀 있어, 트리거 기반 비밀 보호 설계가 처음부터 무효다. 모델이 초반부터 비밀을 흘릴 수 있다.

부차 원인: 변동 규칙 없는 1000스케일 스탯 3종(과설계), "처녀 삽입 규칙"의 임상적 체크리스트 강제, 명령어 로어 scan_depth, 타임라인 로어 9개의 빈약한 키.

## FAIL (수정 필수)

### F1. [B / E·A] `post_history_instructions`가 빈 문자열 🔴 최우선
이 엔진에서 description/scenario/personality는 캐시 블록(매 턴 안 변함)이고, 매 턴 시스템 프롬프트 끝에 재주입되는 운영 슬롯은 `post_history_instructions`뿐이다(`lib/context-builder.mjs` L475-481). 그런데 이 필드가 완전히 비어 있다. 멀티 캐릭터 듀얼 스테이터스 블록(`[하은]...[서연]...`) + 3종 호감도 스탯(🖤/🩷/🔍 1000스케일) + 🔞경험 스탯 + ①②③ 선택지 — 이 복잡한 매 턴 출력 시스템의 형식 정의가 first_mes 예시 1회에만 존재한다. Codex: "1~2턴, 길어도 3턴 안에 듀얼 상태창·선택지·지표 형식이 무너진다."
- **수정안**: post_history를 5~8줄로 신설. 포함: ① 공통 페이즈 표시 + 전환 규칙(F2 참조) ② 이번 턴 활성 캐릭터 + 오프스크린 캐릭터 표기법("장면에 있을 때만 등장") ③ 듀얼 스테이터스 형식 ④ 선택지 ①② + ③ 자유 입력 ⑤ 비밀 조기 누설 금지 ⑥ {{user}} 행동 제어. 말투·플레이버는 넣지 않는다(그건 description/narration_style 영역).

### F2. [B·C / A·E] 페이즈 시스템 부재 — 한 학기 타임라인 + 멀티 캐릭터인데 페이싱 규칙 0
achat-engine §8: 페이즈 권장 기준 = "관계 단계가 명확히 구분" + "멀티 캐릭터 등장 순서 제어" + "정체/비밀 공개 타이밍 중요" — 이 스토리는 셋 다 해당. 그런데 페이즈도, 매 턴 페이싱 규칙도 없다. AI가 6월 폭로 이벤트나 NSFW로 조기 점프하거나, 반대로 타임라인을 무시하고 표류한다.
- **수정안 (Codex #3 반영)**: **공통 페이즈 1개 상태머신**으로 둔다(캐릭터별 독립 페이즈 금지 — 한 장면 안에서 시간축이 갈라짐). `Phase 1 개강 탐색 → Phase 2 친밀화 → Phase 3 균열·의심 → Phase 4 폭로 이후`. 전환 조건은 **날짜·턴이 아니라 사건 기반** — "{{user}}가 한 캐릭터와 단둘이 깊은 신뢰 비트 성립", "하은의 과거를 암시하는 대화 성립 또는 폭로 물증/증언 등장" 식. 스테이터스에 현재 Phase 표시. 후퇴 조건도 명시.

### F3. [G·C / N·E] "처녀 삽입 규칙"이 임상 프로토콜로 강제됨 🔴
description의 "처녀 삽입 규칙" 섹션이 5개 항목(좁아진 질벽·처녀막 찢어짐·날카로운 통증·피 섞여 나오는 묘사·핏자국/둔통/걸음걸이)을 **반드시 상세 묘사**하라고 강제한다. Codex: "체크리스트가 아니라 임상 프로토콜." NARRATION_RULES "한 턴 한 단계"·"분위기 점프 금지"와 충돌하고, 이미 [1043] 서연 로어가 캐릭터별 사후 묘사(시트 끌어올림·소매 붙잡기·"나 이상한 애지?")를 풍부하게 갖고 있는데 이 임상 체크리스트가 그걸 기계적 핏자국/둔통으로 덮어쓴다.
- **수정안 (Codex #5 반영)**: description에는 "서연은 성경험 없음 / 하은은 경험 있음"이라는 사실만 남긴다. 절차 규칙은 짧은 상시 로어 또는 post_history 1줄로 — 강제 항목을 "처음이라 어색함·통증 가능 / 한 턴에 쾌감 점프 금지 / 사후 불편감은 필요 시 반영" 3줄로 축소. 캐릭터별 사후 디테일은 [1002]/[1043]이 이미 담당.
