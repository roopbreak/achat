핵심 설계 원칙

🎯 메인 고급 + 서브 저렴/로컬

메인(③)이 지휘·분해·종합 = 고급 판단

서브(④)가 실무 = 용도별 로컬/저렴 모델. 비용↓·프라이버시↑

🔓 코어 중립 = Claude 비종속

도구·지식(⑤⑥)은 런타임 무관

메인·서브 모두 교체 가능(claude↔codex, 로컬모델 교체). 끊어도 코어 불변

🧮 Ollama는 agent 아님

도구 호출 불가 → 분류·요약·임베딩 서브(TextBackend)로 한정

도구 작업 서브는 claude/codex (Codex 지적 반영)

🔒 보안 = 작업 등급 승인

allowlist는 인증일 뿐

write/delete/send는 디스패처 최종 승인, 외부 콘텐츠=인젝션 입력

예시 — "오늘 일정이랑 관련 노트 정리해줘"

1. 디스패처: 본인 ID 확인 → 작업 등급 read(승인 불필요)
2. 메인(고급): "일정 조회 + 관련 노트 검색 + 요약" 3개 서브태스크로 분해
3. 서브 위임: ① 일정 → calendar MCP(읽기) ② 노트 검색 → 로컬 임베딩 RAG(Ollama, 과금0) ③ 요약 → 로컬 Ollama
4. 메인: 서브 결과 종합·검증 → 텔레그램 회신
→ 고급 모델은 지휘 1회, 검색·요약 대량작업은 로컬에서 무료

🧩 확장성 — 새 용도/기능은 "슬롯에 꽂기"

원칙: 코어 인터페이스는 안정, 추가는 어댑터/레지스트리로. 대부분의 새 용도 = MCP 도구 1개 추가로 끝 → 메인 오케스트레이터가 자동 활용. 코어 코드 수정 0.
