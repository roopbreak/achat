## 1. 밀입국(密入國)이란?

### 정의

유저노트나 페르소나에 **제작자가 의도하지 않은 캐릭터 설정을 강제로 집어넣는 행위**.

- "나 혼자 하는 강제 콜라보"
- 제작자 캐릭터 + 사용자 페르소나 캐릭터 = 혼재 상태
- 예: 무협 세계관에 애니메이션 캐릭터 밀입국, 게임 캐릭터를 현대 학교에 밀입국

### 목적

1. **캐릭터 선호도:** 특정 IP의 캐릭터를 AI 채팅에서 활용
2. **세계관 크로스오버:** 다양한 세계관 조합 탐색
3. **페르소나 상호작용:** 내 캐릭터와 좋아하는 캐릭터의 관계 설정 및 스토리 진행

## 2. 기존 유저노트와의 차이점

### 전통적 유저노트 (기존)

```markdown
{{user}}는 대학생이고, 주술회전 세계관에 산다.
```

- 텍스트 기반 설정만 가능
- 시각적 정보 부재
- 모델이 상상에만 의존

### 밀입국 유저노트 (외부이미지 활용)

```markdown
{{user}}는 원신의 여행자이고, {{char}}와 함께 모험한다.
[외부이미지URL: 여행자 이미지]
```

- **외부이미지 URL** 태그로 캐릭터 비주얼 참조
- 시각적 일관성 확보
- AI가 이미지를 인식하여 보다 정확한 상호작용 생성
- **핵심:** 베이비챗 외부이미지 기능 활용

### 차이점 비교

| 항목 | 기존 유저노트 | 밀입국 + 외부이미지 |
|---|---|---|
| 설정 방식 | 텍스트만 | 텍스트 + 이미지 URL |
| 시각적 일관성 | 없음 | 있음 |
| 캐릭터 인식도 | 낮음 | 높음 |
| 페르소나 명확도 | 모호 | 명확 |
| 외부 세계관 융합 | 어려움 | 쉬움 |

## 4. 밀입국 실전 가이드

### 4.1 밀입국용 캐릭터 이미지 출력 유저노트 작성

**원칙:** 외부이미지 URL + 캐릭터 설정 텍스트 결합

```markdown
{{user}}는 [IP명]의 [캐릭터명]이고, {{char}}와 [상황]이다.

**외형:**
[외부이미지URL]

**성격:** [성격 설명]
**배경:** [캐릭터 배경]
**능력:** [캐릭터 능력/직업]
```

**예시:**

```markdown
{{user}}는 원신의 여행자이며, {{char}}와 함께 티바트 대륙을 모험한다.

**외형:**
https://r2.babechat.com/characters/lumine.webp

**성격:** 온화하면서도 결단력 있는 모험가
**배경:** 다른 세계에서 온 이방인, 오염지를 정화하는 사명
**능력:** 풍원소 조종 및 검술
```

### 4.2 다른 작품에 내 캐릭터 등장시키기

**이브(Eve) / 에덴(Eden) 기반 설정:**

원래 캐릭터를 새로운 세계관에 "캐스팅"하는 방법.

```markdown
{{user}}는 [내 캐릭터명]이고, 
[대상 작품의 세계관]에서 {{char}}와 만난다.

**원작 배경:** [원래 설정]
**현재 상황:** [대상 세계관 내 새 역할]
```

**예시:**

```markdown
{{user}}는 원래 판타지 세계의 검사였지만,
어느 날 갑자기 학원 도시에 나타났다.

블루 아카이브의 학원에 편입되어 {{char}}와 함께 생활한다.
```

### 4.3 자연스럽게 밀입국해서 난입하기

**기본 구조:**

1. 유저노트에서 밀입국 캐릭터의 **배경과 목적** 명시
2. 첫 대사는 **그 목적과 관련된 상황**에서 시작
3. AI가 자연스럽게 세계관 조화를 맞추도록 유도

**예시:**

```markdown
{{user}}는 FATE 세계관의 세이버이고,
어느 날 베이비챗 세계관에 소환되어 {{char}}를 만났다.

서로 처음 보는 존재지만, 뭔가 묘한 운명의 끈을 느낀다.
```

### 4.4 워커스(Worker) 기반 밀입국

플랫폼에서 지원하면:
- 같은 이미지/설정을 여러 챗봇에 밀입국
- 캐릭터 활용도 극대화
- 크로스오버 시나리오 대량 생성

### 4.5 메카물에 탱크 밀입국시키기 (장르 크로스오버)

극단적 예시 (추천 아님):
```markdown
{{user}}는 건담 세계관의 모빌슈트 조종사이지만,
마법소녀 스타일의 변신 메카로 변환되어 {{char}}를 도와준다.
```

## 5. 베이비챗과의 연동 및 활용

### 5.1 베이비챗에서 밀입국 구현하기

**핵심:** 유저노트(User Note) 기능 + 외부이미지 URL

베이비챗은 외부 이미지 URL을 지원하므로, 다음과 같이 구현 가능:

```markdown
# {{user}}의 페르소나

이름: [캐릭터명]
출처: [IP명]

[외부이미지URL]

설정: [상세 성격/배경/능력]
```

**예시:**

```markdown
# {{user}}의 페르소나

이름: 여행자 (루미네)
출처: 원신

https://r2.babechat.com/characters/lumine-genshin.webp

풍원소를 조종하는 모험가.
티바트 대륙의 오염을 정화하는 사명을 가지고 있으며,
언제나 긍정적이고 용감한 마음으로 앞으로 나아간다.
```

### 5.2 캐릭터 일관성 유지 (NAI Char.Ref)

**외부이미지 이미지 생성 워크플로우:**

1. **NAI Studio에서 원하는 캐릭터 이미지 생성**
   - 원본 Character Reference (Char.Ref) 사용
   - 의상, 포즈, 표정 조정

2. **생성된 이미지 확인**
   - 베이비챗 가이드라인 검증 (성기 노출, 가슴 노출 등)
   - 필요 시 인페인트로 수정

3. **R2 업로드**
   - PNG → WebP 변환
   - Cloudflare R2에 업로드
   - 외부 URL 확보

4. **베이비챗 유저노트에 URL 삽입**
   ```markdown
   https://r2.babechat.com/characters/character-name.webp
   ```

### 5.3 밀입국 시 주의사항

- **원작 존중:** 캐릭터의 핵심 성격과 배경은 유지
- **세계관 조화:** AI가 자연스럽게 조화를 맞추도록 설정
- **이미지 품질:** 고해상도 이미지 (512x768 이상) 권장
- **라이선스:** 사용자 개인 용도 범위 내에서만 활용

## 6. 밀입국 관련 용어 정리

| 용어 | 뜻 | 예시 |
|---|---|---|
| **밀입국** | 제작자 의도 외 캐릭터를 유저노트에 추가 | 원신 캐릭터를 블루 아카이브에 밀입국 |
| **페르소나** | 사용자가 역할극할 캐릭터 정체성 | {{user}}의 설정 = 페르소나 |
| **유저노트** | 플랫폼 내 사용자 프로필/설정 텍스트 | 베이비챗의 User Note 기능 |
| **워커스** | 다중 플랫폼 캐릭터 활용 시스템 | 한 캐릭터를 여러 챗봇에 등록 |
| **크로스오버** | 다른 IP/세계관을 혼합 | 나루토 세계관 + 블리치 캐릭터 |
| **외부이미지** | 플랫폼 외부 URL로 참조된 이미지 | https://r2.babechat.com/... |
