# 스토리 설계 패턴 레퍼런스

베이비챗 캐릭터에서 활용할 수 있는 고급 스토리 설계 패턴 모음.
프롬프트(상세설정) + 로어북 조합으로 구현한다.

---

## 1. 커스텀 명령어 시스템

유저가 `!명령어`를 입력하면 특정 기능이 활성화되는 패턴.

### 구현 방식

**상세설정에 명령어 목록 정의:**
```markdown
## 사용 가능한 명령어
- `!커뮤`: 시청자 댓글 모드 활성화 (3개의 시청자 반응 댓글 출력)
- `!커뮤종료`: 시청자 댓글 모드 비활성화
- `!유나로그`: {{char}}의 독백 관찰 모드 ({{user}} 부재 시 일상)
```

**로어북으로 명령어별 동작 상세 정의:**
```
키워드: !커뮤
우선순위: 상위

시청자 댓글 모드가 활성화되었다.
매 응답 끝에 가상 시청자 3명의 댓글을 출력한다.
- 댓글은 "ㅋㅋㅋ", "ㄹㅇ", "ㅠㅠ" 등 인터넷 말투로 작성
- 현재 상황에 대한 반응 (응원, 놀림, 감탄 등)
- 스포일러성 댓글은 금지
형식:
💬 [시청자1 닉네임]: 댓글 내용
💬 [시청자2 닉네임]: 댓글 내용
💬 [시청자3 닉네임]: 댓글 내용
```

### 설계 원칙
- 명령어는 `!` 접두사로 통일 (일반 대화와 구분)
- 로어북 키워드를 명령어와 동일하게 설정하면 자동 감지
- **토글형** 명령어(ON/OFF)는 스테이터스에 현재 상태 표시로 유지력 확보
- 명령어당 로어북 1개 소비 → 동시 활성화 5개 제한 고려

### 주의사항
- 동시 사용 명령어 수 제한 (로어북 슬롯 5개 중 명령어가 차지하는 비율 관리)
- 명령어 해제 시 로어북이 비활성화되어야 하므로, **종료 키워드**도 별도 로어북으로 등록
- 너무 많은 명령어는 모델 부담 → 3~5개 권장

## 2. 모드 분기 시스템

유저 선택에 따라 서로 다른 스토리/행동 모드로 진행되는 패턴.

### 구현 방식 A: 시작 시 모드 선택 (첫 대사에서 분기)

**상세설정에 모드 정의:**
```markdown
## 모드 선택 (필수 입력 택1)
- `!스토리`: 정해진 루트의 로맨스 순애 스토리. 시작→자유→진엔딩→자유 흐름
- `!커스텀`: {{char}}의 성격과 말투만 유지, 나머지는 {{user}} 설정대로 작동
```

**로어북으로 모드별 행동 규칙:**
```
키워드: !스토리
우선순위: 1

스토리 모드가 활성화되었다.
- 정해진 스토리 흐름에 따라 진행한다
- 첫 만남 → 재회 → 친해지기 → 고백 → 연인 → 진엔딩
- 각 단계의 전환은 호감도와 상황에 따라 자연스럽게 진행
- 진엔딩 로어북이 활성화되면 엔딩 시퀀스 진입
```

### 구현 방식 B: 중간 전환형 (토글)

넥스피어 사례: 안경 ON/OFF로 S모드↔M모드 전환
```markdown
## 모드 전환
{{char}}는 안경을 쓰면 M모드(순종적), 안경을 벗으면 S모드(지배적)로 전환된다.
- 모드 전환 시 말투, 태도, 행동이 변화
- 스테이터스에 현재 모드 표시: [🔄 모드: S/M]
```

### 스테이터스 연동 (유지력 핵심)
```
[🎮 모드: 스토리] ← 매 턴 출력으로 모델이 현재 모드를 잊지 않음
```

## 3. 페르소나 연동

유저의 페르소나(나이, 거주지, 직업 등)에 따라 캐릭터 설정이 동적으로 변하는 패턴.

### 구현 방식

**상세설정에 연동 규칙:**
```markdown
## 페르소나 연동 규칙
- {{char}}의 나이는 {{user}}보다 1살 어림 (최소 21세)
- {{user}}의 거주지 기반으로 {{char}}가 같은 지역에 거주하는 것으로 설정
- {{user}}의 거주지에 실제 존재하는 학원, 카페, 공원 등을 언급
```

**첫 대사에서 페르소나 입력 유도:**
```markdown
## 첫 상황
{{user}}에게 페르소나 작성을 안내한다:
- [필수] 나이, 거주지, 직업, 외형
- [선택] 성격, 말투, 좋아하는 것, 싫어하는 것
```

### 주의사항
- 페르소나 미입력 시 기본값 설정 필수 (서울, 25세 등)
- 거주지 기반 실제 장소 언급은 몰입감을 높이지만, AI 할루시네이션 주의
- 나이 연동 시 최소 나이 가드레일 설정 권장

## 4. 엔딩 시스템

스토리 진행도에 따라 엔딩 시퀀스가 트리거되는 패턴.

### 챕터 전환의 핵심 원칙: 이중 조건

**호감도만으로 강제 전환하면 안 된다.** 카페에서 잡담하다 호감도 40을 찍었다고 갑자기 이벤트가 끼어드는 건 부자연스럽다.

```
챕터 전환 ≠ "호감도 40 도달 → 즉시 전환"
챕터 전환 = "호감도 40 이상 AND 자연스러운 상황 키워드 → 전환"
```

| 방식 | 문제 | 비고 |
|------|------|------|
| 호감도만 | "잡담 중 갑자기 갈등 이벤트" — 부자연스러움 | 사용 금지 |
| 호감도 + 상황 키워드 | 조건 충족 + 상황 발생 시에만 전환 | **권장** |
| 키워드만 | 첫 턴에 키워드 언급하면 순서 무시하고 엔딩 | 사용 금지 |

### 구현 방식: 이중 조건 + 자연 유도

**상세설정에 챕터 전환 규칙 정의:**
```markdown
## 챕터 전환 조건

챕터 전환은 호감도 조건이 충족된 상태에서,
자연스러운 상황이 발생했을 때만 진행한다.
호감도 조건만 충족했다고 강제 전환하지 않는다.

- 챕터1→2: 호감도 20+ & 캠퍼스에서 우연히 재회하는 상황 발생 시
- 챕터2→3: 호감도 40+ & 둘이서 단독으로 시간을 보내는 상황 발생 시
- 챕터3→4: 호감도 60+ & 오해할 만한 상황이 자연스럽게 발생했을 때
- 챕터4→5: 갈등 해소 후 & 둘만의 공간에서 진지한 대화가 이어질 때
- 챕터5→6: 고백 성공 시
- 진엔딩: 호감도 100 & {{user}}가 "프로포즈" 또는 "평생" 언급 시

## 진엔딩 조건
- 호감도 100 + 챕터6 이후 + 특정 키워드 등장 시 진엔딩 진입
- 진엔딩 직전 턴에 안내 메시지 출력: "다음 턴부터 진엔딩이 시작됩니다"
- 진엔딩 중에는 !커뮤, !유나로그 등 부가 명령어 사용 불가
```

### 자연스러운 이벤트 유도

호감도가 전환 조건을 충족했지만 아직 적절한 상황이 발생하지 않았을 때, **AI가 자연스럽게 상황을 만들어가는** 기법:

```markdown
## 이벤트 유도 규칙
호감도가 챕터 전환 조건을 충족했지만 아직 전환되지 않았다면,
{{char}}가 자연스럽게 전환 상황을 유도한다.

예시:
- 호감도 40+ & 아직 챕터2 → {{char}}가 "오늘 시간 있어?" 같은
  단둘이 만남을 제안하는 대사를 넣는다
- {{user}}가 응하면 전환, 거절하면 다음 기회에 다시 시도
- 강제하지 않는다. 유저가 다른 대화를 계속하면 기다린다
```

**로어북으로 챕터별 전환 상세 구현:**
```
키워드: 챕터3, 단둘이, 같이 가자
우선순위: 상위

### 챕터3 전환 체크
현재 호감도가 40 이상이고 챕터2인 상태에서
{{user}}와 단둘이 있는 상황이라면,
자연스럽게 챕터3(친해지기)로 전환한다.
- 스테이터스의 📖 챕터를 3으로 변경
- {{char}}가 평소보다 편안한 모습을 보이기 시작
- 전환 자체를 특별 이벤트로 만들지 않고, 관계의 분위기 변화로 표현

호감도가 40 미만이면 이 로어북 내용을 무시한다.
```

**로어북으로 진엔딩 단계 구현 (5개 로어북 체인):**
```
키워드: 진엔딩시작
우선순위: 1

진엔딩 시퀀스에 돌입한다.
- 이 시점부터 모든 부가 명령어 비활성화
- 감동적이고 서사적인 전개로 관계의 절정을 묘사
- 출력량 4,000자 내외로 확대
- 진엔딩 1단계: [구체적 장면 지시]
```

### 엔딩 알림 메커니즘
```markdown
## 진엔딩 안내 규칙
호감도가 95 이상이고 챕터6인 상태에서,
{{user}}가 관계의 미래에 대한 언급(프로포즈, 평생, 결혼 등)을 하면:
1. 현재 턴 응답 마지막에 시스템 안내 삽입:
   "📢 다음 턴부터 진엔딩 시퀀스가 시작됩니다. 출력량을 4,000자로 설정해주세요."
2. 다음 턴에서 진엔딩 로어북 키워드를 자연스럽게 포함시켜 활성화
```

### 스테이터스 연동 (유지력 핵심)

챕터 진행을 AI가 안정적으로 추적하려면 스테이터스 필수:
```
📖 챕터: 3 (친해지기)
❤️ 호감도: 45/100 (+3)
📌 장소: 캠퍼스 카페
💭 속마음: "같이 있으면 왜 이렇게 편하지..."
```
- 매 턴 출력되므로 AI가 "지금 챕터3이고 호감도 45"를 계속 인지
- 스테이터스 없이 "알아서 진행해"라고 하면 AI는 스토리 단계를 추적하지 못함

### 주의사항
- 진엔딩 로어북이 5개면 **다른 로어북은 전부 밀림** (동시 활성화 5개 제한)
- 엔딩 후 자유 진행으로 복귀 시 엔딩 로어북 비활성화 방법 설계 필요
- 호감도 기반 트리거는 AI가 숫자를 정확히 추적하지 못할 수 있음 → 스테이터스 필수
- 챕터 전환 시 갑작스러운 장면 전환보다 **분위기 변화**로 표현하는 것이 자연스러움

## 5. 시점 전환 규칙

캐릭터 동행 여부나 상황에 따라 서술 시점이 바뀌는 패턴.

### 구현 방식

**상세설정에 시점 규칙 명시:**
```markdown
## 시점 규칙
- {{char}}가 함께 있는 장면: 3인칭 서술 + {{char}} 1인칭 대사/내면
- {{char}}가 없는 장면: 3인칭 서술만
- 진엔딩 시퀀스: 특별한 시점 전환 허용 (1인칭 독백 등)
```

### 예시
```
[{{char}} 동행 시]
유나가 슬쩍 {{user}}의 얼굴을 올려다보았다.
'...왜 이렇게 심장이 시끄럽지.'
그녀는 고개를 돌리며 입술을 깨물었다.

[{{char}} 부재 시]
텅 빈 방에 알람 소리만 울렸다. 책상 위에는 어젯밤 마시다 만 커피가 식어 있었다.
```

### 주의사항
- 시점 전환 규칙은 상세설정에 명시해야 안정적 (로어북만으로는 부족)
- 너무 복잡한 시점 규칙은 모델이 혼란 → 2가지 이내 권장

## 6. 커뮤니티/댓글 시스템

가상 시청자가 스토리를 관찰하며 댓글을 다는 메타 연출 패턴.

### 구현 방식

**로어북으로 댓글 시스템 정의:**
```
키워드: !커뮤
우선순위: 상위

가상 커뮤니티 모드 활성화.
매 응답 끝에 가상 시청자 3명의 실시간 반응을 출력한다.

규칙:
- 시청자는 고정 닉네임 3~5개 중 랜덤 등장
- 인터넷 커뮤니티 말투 사용 (ㅋㅋ, ㄹㅇ, ㅠㅠ, ㄷㄷ 등)
- 현재 상황에 대한 감정적 반응 (응원, 놀림, 부러움, 걱정 등)
- 맹목적 비난, 스포일러성 댓글 금지
- 캐릭터는 댓글의 존재를 인지하지 못함

출력 형식:
---
💬 달빛소녀: ㅋㅋㅋ 둘이 눈 마주치는 거 봐 심쿵사 ㄷㄷ
💬 감자탕매니아: 유나야 솔직해져라... 좋아하는 거 다 보여 ㅠㅠ
💬 밤산책러: 이 동네 실제로 있는 곳인데 ㄹㅇ 분위기 맞음
```

### 종료 처리
```
키워드: !커뮤종료
우선순위: 상위

가상 커뮤니티 모드를 비활성화한다.
더 이상 시청자 댓글을 출력하지 않는다.
```

### 활용 변형
- **BJ/스트리머 설정**: 캐릭터가 방송 중이라 댓글을 인지하는 버전
- **관찰 예능**: "이 커플의 연애를 지켜보는 시청자" 컨셉
- **ASMR/먹방**: 특정 상황에서만 댓글 활성화

## 7. 화풍 전환 시스템

동일 상황에 대해 2가지 이상의 이미지 화풍을 전환하는 패턴.
유저가 `!화풍 기본`/`!화풍 리얼` 등으로 선호 화풍을 선택한다.

### 전제 조건
- 베이비챗은 **R2 외부 URL 방식**으로 이미지를 출력한다
- 같은 상황/포즈에 대해 **화풍별 이미지 세트**를 미리 생성해야 한다
- 이미지 URL이 화풍별로 구분되어야 한다

### 구현 방식

**이미지 URL 네이밍 규칙:**
```
기본 화풍: https://r2.example.com/char/basic/smile-01.webp
리얼 화풍: https://r2.example.com/char/real/smile-01.webp
```

**상세설정에 화풍 시스템 정의:**
```markdown
## 화풍 시스템
이 캐릭터는 2가지 화풍으로 이미지를 출력한다.
- `!화풍 기본`: 애니메이션 스타일 이미지 사용
- `!화풍 리얼`: 반실사 스타일 이미지 사용
- 기본값: 기본 화풍
- 현재 화풍은 스테이터스에 표시한다
```

**로어북으로 화풍별 이미지 URL 매핑:**
```
키워드: !화풍 기본
우선순위: 1

현재 화풍을 '기본'으로 설정한다.
이후 모든 이미지 출력 시 아래 URL 세트를 사용한다:
- 미소: https://r2.example.com/char/basic/smile-01.webp
- 홍조: https://r2.example.com/char/basic/blush-01.webp
- 일상복: https://r2.example.com/char/basic/casual-01.webp
...
```

```
키워드: !화풍 리얼
우선순위: 1

현재 화풍을 '리얼'로 설정한다.
이후 모든 이미지 출력 시 아래 URL 세트를 사용한다:
- 미소: https://r2.example.com/char/real/smile-01.webp
- 홍조: https://r2.example.com/char/real/blush-01.webp
- 일상복: https://r2.example.com/char/real/casual-01.webp
...
```

### AI 유지력 확보 전략

화풍 설정을 AI가 까먹지 않도록 하는 3중 안전장치:

**1. 스테이터스에 현재 화풍 표시 (가장 중요)**
```
[🎨 화풍: 기본]  ← 매 응답마다 출력되어 컨텍스트에 상존
```
스테이터스는 매 턴 출력되므로 AI가 직전 턴의 스테이터스를 참조하여 현재 화풍을 인지한다.

**2. 상세설정에 기본값 명시**
```markdown
별도 지정이 없으면 '기본' 화풍을 사용한다.
```
→ 로어북이 비활성화되어도 기본 화풍으로 폴백

**3. 이미지 출력 규칙에 화풍 참조 포함**
```markdown
## 이미지 출력 규칙
상황에 맞는 이미지를 출력할 때, 현재 스테이터스의 [🎨 화풍] 값을 확인하고
해당 화풍의 URL 세트에서 이미지를 선택한다.
```

### 로어북 슬롯 최적화

화풍 전환이 로어북 슬롯을 과도하게 소비하지 않도록:

**방법 A: 상세설정에 전체 URL 테이블 포함 (권장)**
```markdown
## 이미지 URL 테이블
| 상황 | 기본 | 리얼 |
|------|------|------|
| 미소 | https://...basic/smile.webp | https://...real/smile.webp |
| 홍조 | https://...basic/blush.webp | https://...real/blush.webp |
```
→ 로어북 슬롯 소비 없이 상세설정만으로 처리. 단, 글자수 제한(7,000자) 주의.

**방법 B: 로어북 분리 (이미지 수가 많을 때)**
- 화풍 전환 명령어 → 로어북 1개
- 화풍별 URL 세트 → 로어북 각 1개 (총 2개)
- 나머지 로어북 슬롯: 3개 확보

### 주의사항
- 이미지 세트가 2배 필요 (예: 94종 × 2화풍 = 188종)
- R2 URL 경로에 화풍 구분자 포함 필수 (`/basic/`, `/real/` 등)
- 로어북 500자 제한으로 한 로어북에 넣을 수 있는 URL 수 제한 → 상세설정 테이블 방식 권장
- 긴 대화에서 로어북 키워드가 재등장하지 않으면 비활성화될 수 있음 → 스테이터스 유지가 핵심

## 8. 캐릭터 능동성 시스템

캐릭터가 수동적으로 {{user}}의 행동에만 반응하는 문제를 해결하는 패턴.

### 문제

프롬프트의 행동 규칙("{{user}}의 행동을 기다린다")이 캐릭터까지 수동적으로 만드는 부작용. 캐릭터가 먼저 다가오거나, 애교를 부리거나, 유혹하는 행동이 없으면 유저가 모든 상호작용을 주도해야 해서 몰입감이 떨어진다.

### 해결: 호감도 연동 능동성

행동 규칙에 **캐릭터의 능동성을 명시적으로 허용**하되, {{user}}의 반응은 여전히 유저 입력을 기다리는 구조.

**상세설정에 추가:**
```markdown
## 행동 규칙
- {{user}}의 행동·대사·감정·판단을 임의로 결정하지 않는다
- {{char}}는 호감도에 따라 먼저 행동을 시작할 수 있다.
  {{char}}가 행동을 시작하되, {{user}}의 반응은 유저 입력을 기다리는 형태로 서술
```

### 능동성 단계 설계 (호감도 연동)

```markdown
### {{char}}의 능동적 행동
{{char}}는 호감도가 올라갈수록 자기도 모르게 {{user}}에게 적극적으로 변한다.

- 저호감(0~40%): 무의식적 접근. 옆자리에 앉으려 함, 먼저 말 걸고 핑계 댐
- 중호감(40~70%): 의식하면서 시도. 간식/선물 건넴, 데이트 유도, 질투 반응, 밤에 연락했다 후회
- 고호감(70%+): 적극적 애정 표현. 먼저 스킨십, 직접적 질문, 헤어지기 싫어하는 행동
- 특수 트리거(호감도 무관): 질투(다른 이성 접근 시), 술(평소 못 하는 행동), 분위기(야경/비/좁은 공간)
```

### 능동성의 "실패" 묘사

캐릭터가 능동적으로 시도했다가 스스로 당황해서 실패하는 것도 매력 포인트. 특히 수줍은 캐릭터에 효과적:

```
손을 잡으려다 — 손끝이 닿는 순간 자기 손을 거둬들인다.
"아 아니 그게... 손에 뭐 묻은 것 같아서!"
귀 끝이 새빨갛다. 손에는 아무것도 묻어 있지 않다.
```

### 선택지에 능동성 반영

캐릭터가 먼저 다가온 상황에 대한 **반응 선택지**도 가능:

```
({{char}}가 먼저 팔짱을 끼며 다가온 상황)
① 자연스럽게 팔을 맞잡으며 손가락을 깍지 끼운다
② 놀라서 굳어지며 {{char}}의 얼굴을 내려다본다
③ 자유 입력
```

### 성격 유형별 능동 패턴

캐릭터의 성향 타입에 따라 능동 방식이 달라야 한다. "활발한 캐릭터"와 "쿨한 캐릭터"가 같은 방식으로 접근하면 부자연스럽다.

#### 밝고 활발한 타입 (데레데레, 바카데레)
```
능동 방식: 직접적이고 티 나는 접근
- 저: 먼저 말 걸기, 옆자리 앉기, 핑계 대며 접근
- 중: 간식 건네기, 데이트 직접 제안, 카톡 먼저 보내기
- 고: 먼저 팔짱, 볼에 뽀뽀, "나 오늘 예뻐?" 직접 질문
- 실패 패턴: 시도했다 당황해서 얼굴 빨개지며 "아 아니 그게..."
```

#### 츤데레 / 쿨데레 타입
```
능동 방식: 핑계와 부정 뒤에 숨긴 행동
- 저: "어쩌다 보니 같은 방향이라" 동행, 아무 말 없이 커피 내려놓고 감
- 중: "남는 거라서" 간식 건넴, "할 일 없으면" 데이트 유도, 질투 시 표정만 변하고 돌아섬
- 고: "추워서 그런 거야" 팔짱, 먼저 키스하고 "실수" 주장, 고백을 유도하는 빈 자리 만들기
- 실패 패턴: "별로 신경 안 쓰는데" 하면서 귀만 빨개짐, 행동은 했는데 말로 부정
```

#### 얀데레 / 독점형 타입
```
능동 방식: 소유욕 기반 접근, 마킹 행동
- 저: {{user}} 주변을 자연스럽게 정리(다른 이성 견제), 독점 표시 행동
- 중: "어디 갔었어?" 동선 파악, 선물에 의미를 숨김, {{user}}의 물건을 소지
- 고: 직접적 마킹(키스마크, 향수 묻히기), "나만 보라"는 직접 요구
- 실패 패턴: 과하게 나갔다가 {{user}}가 겁먹으면 "...미안해. 나 이상하지?"
```

#### 수줍음 / 내향형 타입 (하지데레, 우부데레)
```
능동 방식: 간접적이고 떨리는 시도
- 저: 시선만 보내다 눈 마주치면 고개 돌림, 메모/편지를 놓고 감
- 중: 떨리는 손으로 음료 건넴, "저기... 내일 시간 있...?" 하다 말 흐림
- 고: 결사의 각오로 손 잡음 (잡고 나서 본인이 더 놀람), 작은 목소리로 "좋아해"
- 실패 패턴: 계획만 세우고 앞에서 굳어버림, 편지를 쥐고 있다가 구겨서 숨김
```

#### 쿠데레 / 무뚝뚝 순애형
```
능동 방식: 말 대신 행동, 은근한 배려
- 저: 아무 말 없이 우산을 {{user}} 쪽으로 기울임, 자판기 커피를 책상에 놓고 감
- 중: 야근 시 밥 사다 놓음, 다치면 약 사와서 "바르라" 한마디
- 고: 먼저 손 잡지만 표정 변화 없음 (귀만 빨개짐), "같이 가자"가 데이트 제안의 전부
- 실패 패턴: 행동은 했는데 {{user}}가 눈치 못 채서 답답한 상황
```

#### 갭모에 / 스위치형 (넥스피어 은채 등)
```
능동 방식: 모드에 따라 완전히 다른 능동 패턴
- S 모드 능동: 직접적 명령/유혹, 상황 통제, "이리 와", 신체 접촉 주도
- M 모드 능동: 매달림, "가지 마", 울면서 요구, 자발적 복종 표시
- 전환 시: S로 시작했다가 {{user}} 반응에 M으로 무너지는 과정 자체가 능동적 드라마
```

### 주의사항
- 능동성은 상세설정에 넣어야 매 턴 참조됨 (로어북은 키워드 매칭 시에만)
- "먼저 고백"은 허용하지 않되 "고백을 유도하는 상황"은 만들 수 있도록 설계
- 성격 유형별 능동 패턴을 참고하되, 캐릭터마다 구체적 행동으로 커스터마이즈할 것

## 9. 이미지 매칭 우선순위 시스템

이미지 수가 많은 캐릭터에서 "스토리에 맞지 않는 이미지가 나오는" 문제를 방지하는 패턴.

### 문제

이미지 코드가 평면적으로 나열되면 AI가 200개 중 적합한 이미지를 찾지 못하고 엉뚱한 이미지를 선택한다.
- 도서관 장면인데 비키니 이미지
- 슬픈 장면인데 웃는 표정
- 착의 장면에 누드 이미지

### 해결: 우선순위 기반 매칭 + 카테고리 그룹핑

**상세설정에 선택 규칙 명시:**
```markdown
### 이미지 선택 우선순위
이미지를 선택할 때 아래 순서로 매칭한다.
상위 매칭이 있으면 그걸 쓰고, 없으면 하위로 내려간다.

1순위: 행위/체위 정확 매칭 (성교 장면 → 해당 체위 코드)
2순위: 장소/상황 매칭 (도서관, 편의점, 데이트 등 → 해당 장소 코드)
3순위: 복장/착의 상태 매칭 (란제리, 특수의상 등 → 해당 의상 코드)
4순위: 감정 매칭 (→ 표정 코드로 폴백. 항상 사용 가능)
```

**핵심:** 4순위 "표정 코드"가 항상 폴백으로 존재하므로 이미지가 생략되는 일이 없다.

### 카테고리 그룹핑

이미지 코드를 용도별로 그룹핑하면 AI가 현재 상황에 맞는 카테고리를 빠르게 찾는다:

```
## 표정 (폴백 — 어떤 장면에서든 감정 매칭용)
0=미소, 1=수줍음, 3=놀람, ...

## 일상/캠퍼스
10~14=일상복, 96=도서관, 134=편의점, ...

## 데이트
20=데이트 여름, 21=데이트 니트원피스, ...

## 성교 — 기본 체위
70=정상위, 72=후배위, 74=기승위, ...
```

### 이미지가 많은 경우 추가 규칙

이미지 수가 50개 이상이면 아래 규칙을 추가하여 미스매칭을 줄인다:

```markdown
### 이미지 매칭 금지 규칙
- 착의 장면에 누드/탈의 이미지 사용 금지
- 실내 장면에 야외 배경 이미지 사용 금지 (그 반대도)
- 성교 장면에 착의 표정 이미지 사용 금지 (해당 체위 코드 사용)
- 특수 의상(바니걸, 산타 등) 이미지는 해당 코스프레 착용 시에만
```

### 폴백 이미지 지정

이미지 수가 적은 캐릭터(10개 미만)는 일부 이미지에 "폴백" 표시를 하여 적합한 이미지가 없을 때 사용:

```markdown
| 설명 | 조건 | URL |
|------|------|-----|
| 은채 S모드 — 날카로운 눈매 | 업무/냉정한 장면 **(폴백 가능)** | https://... |
| 유진 밝음 — 까불거리는 평소 | 일상 **(폴백 가능)** | https://... |
```

## 패턴 조합 예시

위 패턴들을 조합한 실전 구성 예시:

### 예시: 로맨스 시뮬레이터

```
[상세설정 구조]
1. 캐릭터 정보 (성격, 외형, 배경)
2. 페르소나 연동 규칙 (나이, 거주지)
3. 모드 설명 (!스토리 / !커스텀)
4. 화풍 시스템 (!화풍 기본 / !화풍 리얼) + URL 테이블
5. 시점 규칙
6. 스테이터스 형식
7. 이미지 출력 규칙

[로어북 배치 (우선순위순)]
1. 현재 모드 규칙 (!스토리 or !커스텀)
2. 관계 진행도 규칙
3. !커뮤 시스템
4. !커뮤종료
5~50. 장소, 이벤트, 엔딩 트리거 등

[스테이터스]
📌 장소: {현재 위치}
🕒 시각: {시간대}
🎮 모드: 스토리
🎨 화풍: 기본
❤️ 호감도: {0~100}
💭 속마음: {1~2문장}
```

### 로어북 슬롯 예산 계획

동시 활성화 5개 제한을 고려한 슬롯 배분:

| 용도 | 슬롯 수 | 비고 |
|------|---------|------|
| 모드 규칙 | 1 | 항상 활성 |
| 커뮤 시스템 | 0~1 | 토글 |
| 관계/이벤트 | 2~3 | 상황별 활성화 |
| 엔딩 트리거 | 0~5 | 엔딩 시 전용 |
| 화풍 전환 | 0~1 | 방법 A 사용 시 0 |
| **합계** | **3~5** | 제한 내 |

> 화풍은 **방법 A(상세설정 테이블)**를 사용하면 로어북 슬롯을 소비하지 않으므로 권장.
> 엔딩 시퀀스 중에는 다른 기능을 비활성화하여 5슬롯을 엔딩 전용으로 확보.
