### 봇 4: 예 각하! (경영 시뮬 — 최적 사례)
- 컨셉: 국정 운영 경영 시뮬레이션

- **고민**: 방치해도 돌아가도록 vs 게임오버 있는 긴장감
- **결론**: "단순하더라도 밸런스를 잡을 수 있게" — 게임성 유지 선택
- **성과**: 개고수 챗붕이가 만점 찍고 엔딩 달성

### SHN 아키텍처 (참고)
- SHN = 중앙 JSON 상태 객체 (Single Source of Truth)
- 턴 처리 파이프라인:
  1. 상태 업데이트 (물리/건강 등 시뮬 엔진 호출)
  2. 데이터→텍스트 변환
  3. 서사 생성 (내러티브 모듈)
  4. 선택지 생성
  5. 최종 UI 렌더링
- 모듈간 느슨한 결합 → 안정성 확보

## 5. 장기 기억 활용 시스템

출처: arca.live/b/characterai/159376399 (장기 기억 활용 시스템)

### 핵심 개념
- 연상 기억(Associative Memory): 컨텍스트 유사성에 의해 자동 검색되는 부분적 기억
- δ(델타): 즉각적인 변화 — 본 장면에서 발생, 반응을 유발
- δ_Fermented: 누적된 영향 — 장면 단위로 과거 기억에 귀속
- 습관적 관성: 장기적 패턴 — 단일 사건으로 변하지 않음
- 관계적 예치금: 누적된 상호작용 내역

### 상태 필드 포맷
```
<TELESCOPE_STATUS>
<Info>장소 | 날짜=YYYY-MM-DD | 시간=HH:mm</Info>
<δ>이름="영향(+-n)","영향(+-n)"│이름2="영향(+-n)"│...</δ>
</TELESCOPE_STATUS>
```

### δ 필드 규칙
- 출력 조건: 유의미한 심리적 변화 발생 시
- 영향 라벨: 감정 또는 가치관 (신뢰/불안/호감/분노/죄책감/희망 등)
- 강도: 1-3=표면적 / 4-6=현저함 / 7-9=심층적/근본적

### 인지 처리 규칙
- 과거: 관련성에 비례하여 기억이 자동 활성화, 불변
- 현재: 유일한 행위 지점
- 흐름: 자극 → 앵커의 감각 → 지각 → 반응

## 7. 시뮬봇 설계 핵심 원칙 (커뮤니티 합의)

### DO (해야 할 것)
1. 턴 제한을 반드시 설정 (20턴 권장)
2. 매 턴 스테이터스/상태창 출력 필수
3. 선택지 항상 제공 (인풋 장애 대응)
4. 단순하게 시작, 점진적 확장
5. NPC는 간략하게 (1캐릭 150자 이내)
6. 밸런스 테스트 필수 (제작자가 직접 플레이)
7. 스코어/엔딩 시스템으로 리플레이 유도
8. 에셋 없이도 시스템+컨셉으로 충분 (치한열차 사례)

### DON'T (하지 말 것)
1. 시스템만 계속 추가하면서 밸런스 무시
2. 캐릭터 30명에 상세 시트 (토큰 폭발)
3. 40턴 이상 장기 시뮬 (AI가 상태 추적 못함)
4. 에셋+상태창+선택지+이미지 동시 출력 (포맷 충돌)
5. 방치해도 진행되는 무긴장 설계 (게임성 상실)
6. 수치 정밀 계산 의존 (AI가 못함, 단계제 사용)
