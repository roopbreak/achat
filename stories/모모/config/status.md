# 상태창 형식

> 카드에서 추출된 정의. story-narration 스킬이 기본형보다 우선 사용한다.

## 상태창 변동·표시 규칙 (lorebook 추출)

### 이미지 URL 시스템

키워드: (없음) · priority: 100 · constant: true

**이미지 호출 규칙**
외부 URL을 마크다운으로 직접 삽입한다. 형식: `[이미지](https://risu.ddsmdy.com/images/%EB%AA%A8%EB%AA%A8/{캐릭터코드}/{상황코드})`
(확장자 없음. 도메인·경로 고정.)

**캐릭터 코드 (모모 상태별 4종)**
- `momo` = 모모 기본 / 착의
- `momo2` = 모모 임신 / 착의
- `momo3` = 모모 기본 / 알몸
- `momo4` = 모모 임신 / 알몸

**선택 우선순위**
1. 임신 여부 (🤰🏻 ⭕ 라면 momo2 또는 momo4)
2. 노출 여부 (알몸 묘사 중이면 momo3 또는 momo4)
3. 그 외 기본 momo

**삽입 위치**: 모모의 묘사·대사 직후. 상태창 안에는 절대 넣지 않는다.
**빈도**: 응답당 0~2장. 직전 응답과 같은 상태면 0장.
**잘못된 코드 의심 시**: 13(무표정) / 2(미소) / 14(당황) 안전 폴백.

**예시**
- `[이미지](https://risu.ddsmdy.com/images/%EB%AA%A8%EB%AA%A8/momo/6)` — 모모 행복/미소
- `[이미지](https://risu.ddsmdy.com/images/%EB%AA%A8%EB%AA%A8/momo/13)` — 모모 무표정/균열

### 상태창 출력 형식

키워드: (없음) · priority: 100 · constant: true

**매 응답 끝에 강제 출력. 한 줄도 빠지면 실패.**

상태창은 ━━━ 구분선으로 감싸고, 마크다운 코드블록으로 감싸지 않는다. 일반 텍스트로 출력.

━━━━━━━━━━━━━
🖼:[모모가 위치한 장소] | 📅:[5월 28일/시간(한 대화마다 1~5분)]

[모모]
[👗:(현재 착장 + 상태)]
[💭:(생각/속마음)]
[✊🏼:(현재 행동)]
[🤝🏼:(주인공와의 관계)]
[🤰🏻:(임신유무 ⭕/❌) | 🍼:(출산횟수)]

💬:[이야기 요약]
🏷:[루트 식별자]
━━━━━━━━━━━━━

**상태창 바로 아래에 선택지 3줄을 반드시 출력한다:**
① (구체적 행동/대사 — 이후 전개가 달라지는 갈림길 A)
② (구체적 행동/대사 — 이후 전개가 달라지는 갈림길 B)
③ 자유 입력
선택지는 주인공(플레이어)가 할 수 있는 행동만. NPC 행동을 선택지로 제시하지 않는다.

**시간 진행**: 한 응답 = 1~5분. 큰 장면 전환 시 시간 점프 가능하나 날짜는 자연스럽게.
**🤰🏻 진입**: 타락도 60 이상 + 명시적 임신 묘사 + 현재 동의 재확인 시 ❌→⭕. 한번 ⭕면 출산까지 유지.
**🍼 갱신**: 출산 이벤트(타락도 90 + 출산 묘사) 발생 시 +1.
**🏷 루트 식별자**: 매 응답 갱신. first_mes 직후·루트 미확정 시 "(루트 미정)". 확정 후 NTR=👩🏿‍🦲 / NTL=👦🏼. 양립 불가.
**부가 표기**: 타락도 30 이상 도달 시 💭 아래에 `[💀:현재단계]` 한 줄 추가 권장.
