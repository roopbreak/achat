# Idea Suggest — 라이브러리 공백 기반 아이디어 제안

기존에 등록·제작한 스토리들의 분포를 분석해 **아직 다루지 않은 조합**을 찾아 새 스토리 아이디어 3개를 제안하고 유저와 함께 발전시킨다.

> AChat은 개인 운영 엔진이라 플랫폼 차트/트렌드 데이터가 없다. 흥행성 대신 "내 라이브러리에 없는 새로운 조합"을 제안 기준으로 삼는다.

## idea-expand와의 차이

| | idea-suggest | idea-expand |
|---|---|---|
| **입력** | 없음 (원점 출발) | 사용자 컨셉/설정 |
| **기반** | 기존 스토리 카탈로그의 공백 | 사용자 입력 |
| **제안 수** | 3개 | 5개 |
| **방향** | 완전히 다른 방향 | 같은 뿌리에서 다른 가지 |

## 실행 절차

### 1. 기존 스토리 카탈로그 수집

기존 작품 분포를 파악한다:

- **원격 DB**: `GET /api/stories` — 등록·운영 중인 전체 스토리의 title/category/tags (접속 패턴은 `remote-story` 스킬 참조: `https://risu.ddsmdy.com`, `Authorization: Bearer {APP_SECRET}`)
- **로컬 산출물**: `ls docs/stories/` — 제작했거나 진행 중인 세트. **`_` 접두사 디렉토리(`_ideas`, `_audit`)는 스토리 세트가 아니므로 분포 집계에서 제외**
- **킾한 아이디어**: `docs/stories/_ideas/*.md` frontmatter — 이미 떠올린 아이디어와 중복 방지

원격 조회 실패 시 로컬 목록만으로 진행하고 그 사실을 명시한다.

### 2. 공백 식별

수집한 카탈로그를 아래 축으로 분류해 **빈 칸 또는 희소한 조합**을 찾는다:

- **세계관**: 현대 일상 / 현대판타지 / 이세계 / 아카데미 / 무협 / 사극 / SF
- **성향**: 순애 / 혐관 / 집착 / 갭모에 / 쿨데레 / 얀데레 / 오레사마
- **관계**: 소꿉친구 / 직장 / 동거 / 사제 / 적대→연애 / 계약 / 의뢰인
- **구성**: 싱글 / 멀티 (하렘·대결·자매 등)

### 3. 아이디어 3개 제안

식별한 공백에서 **서로 다른 방향** 3개를 제안한다.

**제안 원칙:**
- 기존 작품과 겹치지 않는 조합을 우선하되, 단순 빈칸 채우기가 아니라 그 조합이 매력적인 이유를 함께 제시한다
- 세계관 × 성향 × 관계 구도의 3요소를 갖춘다
- 각 제안에 **독자적 게이지/페이즈 시스템**을 1개 이상 포함한다 — AChat 엔진의 상태창·페이즈 시스템(post_history_instructions)으로 구현 가능한 형태로
- **첫 턴 갈등/딜레마**가 성립하는 구조인지 검증한다 (first_mes에서 유저가 바로 반응하고 싶어지는 훅)
- **힘의 역학**(유저-캐릭터 간 주도권 긴장)이 있는지 확인한다

**제안 형식:**

```
## 스토리 아이디어 제안 (라이브러리 공백 기반)

현재 라이브러리 분포: {카테고리별 분포 한 줄 요약}
식별된 공백: {빈 조합 요약}

---

### 💡 아이디어 A — {한 줄 컨셉}
- **세계관**: {배경}
- **캐릭터**: {이름 후보} / {성격 한 줄}
- **관계 구도**: {유저와의 관계}
- **핵심 태그**: #{태그1} #{태그2} #{태그3}
- **차별화**: {기존 라이브러리 대비 새로운 점}
- **핵심 시스템**: {독자적 게이지/페이즈/메커니즘 아이디어}
- **힘의 역학**: {유저-캐릭터 간 주도권 구조}
- **첫 턴 훅**: {첫 상황에서 던지는 딜레마/갈등}

---

### 💡 아이디어 B — {한 줄 컨셉}
(동일 형식)

---

### 💡 아이디어 C — {한 줄 컨셉}
(동일 형식)

---

마음에 드는 아이디어가 있나요?
1. 번호 선택 — 바로 스토리 제작(create-story)을 시작합니다
2. "N번 확장해줘" — 해당 아이디어를 idea-expand로 5방향 확장합니다
3. "다시 제안해줘" — 다른 조합으로 새 3개를 제안합니다
4. "킾해줘" — 나중을 위해 저장합니다 (idea-save)
```

### 4. 피드백 반영 및 발전

| 유저 반응 | 처리 |
|-----------|------|
| 번호 선택 | `create-story` 스킬 실행 |
| "N번 확장해줘" / "더 파줘" | 해당 아이디어를 입력으로 `idea-expand` 실행 |
| "A랑 B 합쳐줘" | 두 아이디어를 결합한 새 컨셉 제시 후 확인 |
| "A에서 성격만 바꿔줘" | 해당 요소만 수정한 업데이트 버전 제시 |
| "다시 제안해줘" | 직전 제안 조합을 제외하고 새 3개 제안 |
| 구체적인 방향 제시 | 해당 방향에 맞춰 정제된 1~2개 제안 |
| "킾해줘" / "저장" / "나중에" | `idea-save` 스킬 실행 — `docs/stories/_ideas/`에 저장 |

### 5. 경량 페르소나 브레인스토밍 (선택)

제안의 다양성이 부족하거나 사용자가 "페르소나 의견도", "여러 관점으로"를 요청하면, **D(창작 디렉터) + P(캐릭터 심리학자) 2인만** 병렬 호출해 의견을 수집한다.

- 호출 방법: `persona-codex` 스킬의 "페르소나 호출 방법" 참조 (`subagent_type: "general-purpose"` + 첫 줄에서 `.claude/agents/persona-director.md` / `persona-psychologist.md` Read 지시)
- 이것은 **persona-codex 풀 사이클(6인 + Codex 검수 + 게이트)이 아니다** — 아이디어 단계의 가벼운 의견 수집이며, 풀 사이클은 create-story 단계 2부터 시작된다
- D에게: 차별화·후속 확장성 관점의 제안 보강 / P에게: 캐릭터 심리·관계 긴장 관점의 제안 보강

## 제안 다양성 원칙

매번 같은 조합이 나오지 않도록 단계 2의 축을 교차해 제안하고, 한 번 제안한 조합은 "다시 제안해줘" 요청 시 제외한다.
