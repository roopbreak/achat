# /st-new

새 스토리를 생성한다. `stories/_template/` 을 복사하고 인터뷰로 핵심 항목을 채운다.

ARGUMENTS: `$ARGUMENTS`

## 실행

### 1. 스토리명 결정

- ARGUMENTS 첫 토큰을 디렉토리명으로 사용 (한글/영문 모두 허용)
- 없으면 사용자에게 묻기
- `stories/{스토리명}/` 이미 존재하면 덮어쓸지 확인

### 2. 템플릿 복사

```bash
cp -r stories/_template stories/{스토리명}
```

### 3. 인터뷰 (사용자에게 한 번에 묻기)

다음 5가지를 한 응답에서 모두 질문:
1. **장르 + 배경 (한 줄)** — 예: "현대 고등학교 순정 로맨스", "사극 무협"
2. **메인 캐릭터 이름 + 한 줄 컨셉** — 예: "강서윤, 32세 마케팅 팀장, 일에 미친 워커홀릭"
3. **주인공 설정** — 예: "27세 신입사원, 강서윤의 부하"
4. **첫 장면 (한 단락)** — 예: "회식 후 단둘이 남은 오피스, 자정"
5. **성적 서술 스타일 / 수위** — 예: "수위 최대, 직접적 호칭 사용" 또는 "은유 위주"

### 4. 변환 적용

답변을 받아 다음 파일을 생성/갱신:
- `stories/{이름}/config/context.md` — 세계관 + 캐릭터 + 규칙
- `stories/{이름}/config/notes.md` — 화이트리스트 + 절대 규칙
- `stories/{이름}/intro.md` — 첫 장면
- `stories/{이름}/config/lorebook/characters.md` — 메인 캐릭터 프로필
- `stories/{이름}/config/lorebook/index.md` — 비어 있는 인덱스 (사용자가 추후 추가)

### 5. 별칭 등록

`.playdata/story-aliases.md` 테이블 끝에 새 행 추가. 별칭은 사용자에게 묻거나 빈 칸으로 남김.

### 6. 안내

"새 스토리 `{이름}` 생성 완료. `/st-continue {이름}` 으로 시작할 수 있습니다." 출력.

## 주의

- 상태창은 비워둠 (필요 시 사용자가 `config/status.md` 에 카드별 형식 추가)
- 선택지 규칙은 기본 미적용 (사용자가 원하면 status.md 에 ## 선택지 규칙 추가)
- 인터뷰 결과는 최대한 간결하게 변환 — 후속에 `/st-continue` 시작하면 즉시 채팅 가능하도록
