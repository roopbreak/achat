# Idea Save — 아이디어 킾(Keep)

현재 대화에서 논의 중인 스토리 아이디어를 `docs/stories/_ideas/`에 저장하여 나중에 이어서 작업할 수 있게 한다.

## 저장 위치

`docs/stories/_ideas/{slug}.md`

- slug: 아이디어 핵심을 영문 kebab-case로 (예: `vampire-cafe`, `twin-rivalry`) — 이후 `create-story` 진행 시 스토리 디렉토리명·DB slug로 그대로 이어지므로 가이드(`docs/guide/story-creation-webengine.md` §2)의 slug 규칙(`^[a-z0-9][a-z0-9-]{2,49}$`)을 따른다
- `docs/stories/` 하위라 git 추적 대상 — 산출물을 git에 남기는 프로젝트 관례를 따른다 (`_` 접두사 디렉토리는 스토리 세트가 아닌 보조 디렉토리: `_audit`, `_ideas`)
- ⚠️ `docs/stories/`를 스토리 목록으로 순회하는 스킬·스크립트는 **`_` 접두사 디렉토리를 반드시 제외**해야 한다 (idea-suggest 단계 1에 반영됨)

## 실행 절차

### 1. 현재 대화 컨텍스트 수집

대화에서 논의된 아이디어 정보를 아래 템플릿 항목 기준으로 정리한다. 없는 항목은 `미정`으로 표시한다.

### 2. 파일 생성

`docs/stories/_ideas/{slug}.md`로 저장한다. 본문 구조는 create-story 단계 1이 파악하는 항목(장르/캐릭터/세계관/구성/옵션)과 맞춰져 있어 **단계 1의 기반 입력으로 사용**할 수 있다 (단계 1에 별도 파서는 없다 — 자유형 입력 정리이므로 미정 항목은 게이트에서 확인받는다).

```markdown
---
name: {아이디어 이름 (한글)}
slug: {slug}
created: {YYYY-MM-DD}
status: draft
source: {idea-expand | idea-suggest | direct}
tags: [{태그1}, {태그2}, ...]
---

# {아이디어 이름}

## 컨셉 요약
{1~3줄 핵심 요약}

## 장르
{장르/카테고리. AChat category 후보: 현대 로맨스 / 판타지 / 사극 / 무협 등}

## 캐릭터

### 메인: {이름}
- **성격**:
- **외모**:
- **배경/직업**:
- **유저와의 관계**:

### 서브: {이름} (멀티 구성일 때만)
(동일 형식)

## 세계관/배경
{세계관 설명. 미정 시 "미정"}

## 핵심 갈등/매력
{스토리를 끌고 갈 핵심 요소}

## 핵심 시스템 (게이지/페이즈)
{독자적 게이지·역치·페이즈 아이디어. 미정 시 "미정"}

## 구성/옵션
- 구성 모드: {싱글 / 멀티 (서브 N명) / 미정}
- 스테이터스: {사용 / 미사용 / 미정}
- 성인 콘텐츠: {포함 / 미포함 / 미정}
- 이미지: {NAI 연동 (기본) / 미정}

## 대화 메모
{논의 중 나온 중요 결정사항이나 방향성 메모}
```

### 3. 저장 확인 출력

```
아이디어 저장 완료!

- 파일: docs/stories/_ideas/{slug}.md
- 상태: draft
- 이어서 하려면: "{아이디어 이름} 이어서 해줘" — create-story 단계 1부터 시작합니다
```

## 상태값

| 상태 | 의미 |
|------|------|
| `draft` | 아이디어만 있는 상태. 컨셉 확정 전 |
| `concept-done` | 컨셉 방향 확정, 제작 대기 |
| `in-progress` | create-story 진행 중 |
| `completed` | 제작 완료 (DB 등록까지) |
| `abandoned` | 폐기 |

## 재개 흐름 ("이어서 해줘")

사용자가 저장된 아이디어의 재개를 요청하면:

1. `docs/stories/_ideas/`에서 해당 파일을 찾아 Read (이름이 모호하면 목록 제시 후 선택)
2. frontmatter `status`를 `in-progress`로 갱신
3. `create-story` 스킬 단계 1로 진입 — 아이디어 본문을 `docs/stories/{slug}/00_input.md`의 기반으로 사용 (스토리 디렉토리명 = 이 파일의 `slug`. create-story 문서의 `{name}` 표기와 같은 값이다. 미정 항목은 단계 1 게이트에서 사용자에게 확인)
4. create-story 단계 5(DB 등록)까지 완료되면 `status: completed`로 갱신

"아이디어 목록 보여줘" 요청 시: `docs/stories/_ideas/*.md`의 frontmatter(name/status/created/tags)를 표로 제시한다.

## idea-expand / idea-suggest에서의 연결

아이디어 확장/제안 후 아래 반응을 보이면 이 스킬을 실행한다:

| 유저 반응 | 처리 |
|-----------|------|
| "킾해줘" / "저장" / "나중에" / "보관" | 논의 중인 아이디어 저장 |
| "N번 킾해줘" | 해당 번호 아이디어만 저장 |
| "다 킾해줘" | 제안된 모든 아이디어를 개별 파일로 저장 |
