# 자동 저장 절차 (7회 교환)

## 세이브 슬롯 경로 규칙

모든 저장 경로는 `{슬롯경로}/`를 기준으로 한다. `{슬롯경로}`는 CLAUDE.md의 "세이브 슬롯 경로 해석 규칙"에 따라 결정된다.

| 구 경로 | 신 경로 |
|---------|---------|
| `.playdata/{스토리명}/sessions/session_NNN.md` | `{슬롯경로}/sessions/session_NNN.md` |
| `.playdata/{스토리명}/status.md` | `{슬롯경로}/status.md` |
| `.playdata/{스토리명}/summaries/summary_NNN.md` | `{슬롯경로}/summaries/summary_NNN.md` |
| `.playdata/{스토리명}/summaries/story_so_far.md` | `{슬롯경로}/summaries/story_so_far.md` |
| `.playdata/{스토리명}/summaries/story_archive.md` | `{슬롯경로}/summaries/story_archive.md` |
| `.playdata/{스토리명}/relationship.md` | `{슬롯경로}/relationship.md` |

---

교환 횟수가 15의 배수에 도달하면, **해당 응답에서는 서술만 출력**한다. 저장은 전부 **다음 턴 입력 시** 백그라운드로 실행된다.

**⚠️ 핵심 원칙: 저장이 서술을 블로킹하지 않는다. 모든 파일 저장은 `run_in_background: true` Bash로 실행한다.**

---

## 15교환째 응답

1. 서술을 정상 출력한다
2. 상태창에 `[교환: 15/30(저장)]`을 표시한다 (다음 저장 시점 업데이트)
3. **파일 저장을 하지 않는다**
4. 내부적으로 "다음 턴에 저장 실행" 플래그를 기억한다

## 16교환째 (다음 턴) — 저장 + 서술 동시 실행

사용자 입력 수신 시, **하나의 응답에서** 아래를 모두 동시에 보낸다:

### Phase 1 — 즉시 백그라운드 (Bash `run_in_background: true`)

분석 불필요. 컨텍스트에 있는 데이터를 그대로 파일에 쓴다.

**1-A. 세션 파일 append:**
```bash
cat << 'EOF' >> {슬롯경로}/sessions/session_NNN.md

<!-- 교환 N | YYYY-MM-DD HH:MM | 작중: M월 D일(요일) HH:MM -->
{이전 턴(7교환째) 서술 원문}
EOF
```

**1-B. 상태값 저장:**
```bash
cat << 'EOF' > {슬롯경로}/status.md
{현재 상태창 전체 내용}
EOF
```

### Phase 2 — 분석 후 백그라운드 (Bash `run_in_background: true`)

요약·관계 분석은 AI 추론이 필요하지만, **결과물을 Bash heredoc으로 파일에 쓰는 것**은 백그라운드로 가능하다.

분석을 먼저 수행한 뒤, 3개 파일을 각각 `run_in_background` Bash로 저장한다.

**2-A. 세션별 요약:**
```bash
cat << 'EOF' > {슬롯경로}/summaries/summary_NNN.md
# 요약 NNN

> 날짜: YYYY-MM-DD
> 교환 범위: N회 ~ M회

## 주요 사건
- 사건 1
- 사건 2

## 현재 상황
(한 줄 요약)

## 관계 변화
- {캐릭터A} ↔ {캐릭터B}: 변화 내용

## 미결 복선
- 복선 1: 상태
EOF
```

**2-B. 누적 요약 업데이트:**
```bash
cat << 'EOF' > {슬롯경로}/summaries/story_so_far.md
{기존 내용 + 이번 요약 병합 결과. 4,000자 이내}
EOF
```
- 4,000자 초과 시: 오래된 사건을 1줄 압축하여 `{슬롯경로}/summaries/story_archive.md`에 별도 append
- **복선 보존 원칙**: 미해결 복선·떡밥은 아무리 오래되어도 story_so_far에서 삭제하지 않는다. 해결된 복선만 압축 대상.

**2-C. 관계·복선 트래커 저장:**
```bash
cat << 'EOF' > {슬롯경로}/relationship.md
{전체 관계 + 복선 + 정보격리 데이터}
EOF
```

relationship.md 필수 섹션:

```markdown
# 관계·복선 트래��: {스토리명}
> 최종 업데이트: YYYY-MM-DD (교환 N)

## 관계
{캐릭터 간 감정 벡터, 호칭, 거리감, 터닝 포인트}

## 미결 복선
- [교환 N] {복선 내용} — 상태: 미해결/진행중
- [교환 N] {복선 내용} — 상태: 미해결
{해결된 복선은 삭제하고 story_so_far에 결과만 기록}

## 약속·맹세
- [교환 N] {��가 → 누구에게}: "{내용}"
{파기된 약속은 표시: ~~파기~~ + 이유}

## 정보 격리 (누가 뭘 아는가)
- {캐릭터A}: {알고 있는 비밀/사실} | 모르는 것: {모르는 핵심 정보}
- {캐릭터B}: ...
{AI가 캐릭터에게 잘못된 정보를 부여하지 않기 위한 체크리스트}

## 캐릭터 현재 상태
{복장, 위치, 신체/정서 상태 — 다음 턴 이어가기 위�� 스냅샷}
```

### 서술 출력

Phase 1, Phase 2의 Bash 호출과 **동시에** 서술 텍스트를 출력한다. 저장 완료를 기다리지 않는다.

---

## 주의사항

- **저장 완료 메시지를 출력하지 않는다.** 사용자는 저장을 의식하지 않고 이어서 플레이한다.
- Phase 2 분석은 서술 생성과 동시에 AI 내부에서 수행한다. 파일 쓰기만 백그라운드.
- story_so_far.md 업데이트 시 기존 파일을 Read해야 하는 경우, 서술 출력 전에 Read를 먼저 호출한다.
- `/st-save` 수동 저장 시에도 동일한 절차를 따르되, **즉시 실행**한다 (다음 턴으로 미루지 않음).
- 세션 번호는 기존 파일 개수 + 1로 자동 산출
- 저장 실패 시 재시도 1회, 그래도 실패 시 사용자에게 수동 저장 안내
