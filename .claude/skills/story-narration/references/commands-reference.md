## 스토리 이름 해석 규칙

모든 커맨드에서 스토리명을 인자로 받을 때, 아래 세 가지 방식을 모두 지원한다.

1. **번호**: `.playdata/story-aliases.md` 테이블의 `#` 열 (예: `3` → 미래에서_온_너)
2. **별칭**: 같은 테이블의 `별칭` 열 (예: `pr` → 풀리퀘스트)
3. **원래 이름**: `stories/` 디렉토리명 그대로 (예: `풀리퀘스트`)

**해석 순서**: 숫자이면 번호 → 아니면 별칭 매칭 → 아니면 디렉토리명 매칭 → 부분 매칭 시도 → 실패 시 목록 표시 후 선택 요청.

인자가 없으면 목록을 보여주고 번호로 선택하게 한다.

- `/st-new` 실행 시: `.playdata/story-aliases.md`에 행 추가 (별칭은 AI가 짧은 영문 약어 제안)
- 스토리 삭제 시: 해당 행 제거
