# AChat 스토리 QA 체크리스트

검수 모드에서 페르소나 팀이 약점을 발굴할 때 쓰는 객관적 점검 기준.
페르소나의 주관적 렌즈(차별화·매력·욕망 톤 등)와 **별개로**, 이 체크리스트는 엔진 호환성·구조 정합성 같은 **판정 가능한** 항목을 다룬다.

검수 산출물에서 각 항목을 PASS / WARN / FAIL로 표기한다.

## 담당 페르소나 매핑

| 체크 영역 | 주 담당 | 보조 |
|----------|---------|------|
| A. 변수 검증 | A 채팅작가 | — |
| B. 프롬프트 구조 | A 채팅작가 | D 디렉터 |
| B2. 규칙 예산 | **과잉=E 엔진QA / 정체성핵=P 매력설계자** (항목별 단독 판정) | — |
| C. 엔진 호환성 | A 채팅작가 | — |
| D. 로어북 검증 | A 채팅작가 | K 컬처(키워드 적절성) |
| E. 일관성 검증 | D 디렉터 | P 매력설계자, K 컬처 |
| F. 캐릭터 품질 | P 매력설계자 | D 디렉터 |
| G. NSFW 톤·용어 | N NSFW작가 | — |
| H. 대화 시뮬레이션 | A 채팅작가 | 전원 |

## A. 변수 검증 (FAIL 기준)

- [ ] `{{char}}`, `{{user}}` 이중 중괄호 사용 — `{char}`/`{user}` 단일 괄호 발견 시 **FAIL**
- [ ] `{{char}}` 치환 후 문맥이 자연스러운가
- [ ] description·로어북 본문에서 캐릭터 실제 이름을 쓰는 게 더 안전한 자리에 변수를 남발하지 않았는가

## B. 프롬프트 구조 (WARN/FAIL)

| 체크 | FAIL | WARN |
|------|------|------|
| 정보/지시 분리 | 완전히 뒤섞임 | 부분 혼재 |
| 구체적 지시 | 전체가 모호 | 일부 모호 |
| 동시 시스템 과다 | 3개 이상 동시 요구 | 2개 동시 |
| 서술 품질 지침 | 없음 | 불충분 |
| post_history 길이 | — | 5줄 초과로 흘려보낼 위험 |

## B2. 규칙 예산 (양방향 — 규칙 과부하 시 AI가 런타임 톤 규칙을 흘림)

> 카드 규칙 총량이 AI 한계를 넘으면 AI가 일부를 흘리는데, 가장 먼저 희생되는 게 런타임(`lib/prompt/builtins.mjs`)의 공통 톤 규칙(천박·직설)이다 → 결과적으로 점잖아짐. 원칙: **"절차 규칙은 줄이되 정체성 핵은 진하게."** 단순 총량 삭감이 아니다(핵을 빼면 평균회귀로 차별화 붕괴).

**카운트 단위**: "독립적으로 실패 가능한 행동 제약" 1개 (문장 수가 아님 — 한 문장에 제약 2개면 2개, 형식 지시 묶음은 1개). 임계값 N·M은 파일럿 실측으로 확정하되, 실측 전 **예시 10개를 먼저 고정**해 카운트 기준을 캘리브레이션한다.

| 체크 | FAIL | WARN | 단독 판정권자 |
|------|------|------|--------------|
| (과잉) 절차 의무 규칙 수 | M개 초과 | N개 초과 | **E** |
| (과잉) 런타임 중복 재서술 | — | builtins가 이미 부과하는 톤·분량·상태창·서술 규칙을 카드가 재서술 | **E** |
| (과잉) 우선순위 없는 평면 나열 | — | 임계 근처인데 "핵심 3개" 표시 없음 | **E** |
| (누락) 정체성 핵 흐릿 | 핵이 사실상 없음 | 욕망·태도·말투 핵 / 고유설정 / 세계관 제약 / NSFW 로어 중 누락 → 평균회귀 위험 | **P** |

> 판정권자 단일화: 과잉 3항목은 E 단독, 정체성 핵 누락은 재편 P 단독. 공동 담당 금지(책임 분산 방지).

## C. 엔진 호환성 (FAIL 기준)

- [ ] 출력 분량 직접 지정("500자 이상") 없음 — maxTokens로 제어됨
- [ ] 이미지 키워드 방식(`img: 표정`) 미사용 — scene_key 시스템
- [ ] 자주 변하는 내용을 description에 넣지 않음 — prompt-caching 효율
- [ ] 턴 수 기반 조건("10턴 후") 없음 — AI는 턴을 못 셈
- [ ] JSON 포맷 미사용
- [ ] 서술 명령어(`~행동~` 등) 재정의·충돌 없음

## D. 로어북 검증 (WARN/FAIL)

| 체크 | FAIL | WARN |
|------|------|------|
| 키워드 형태 | 자연어 구절 사용 | 일상어·범용어 사용 |
| 항목 수 | 50개 초과 | 30개 이상 |
| 우선순위 | 핵심 정보가 저우선순위 | 배치 기준 불명확 |
| 중복 | description과 완전 중복 | 부분 중복 |
| 상시 로어 수 | 4개 이상 | 3개 |
| 토큰 예산 | 동시 활성 시 예산 초과 가능 | 근접 |
| 상시 규칙 배치 | 상시 규칙인데 constant=0 (구두점 트리거 의존) | — |
| 명령어 트리거 | `!command` 인데 scan_depth≠1 | — |

## E. 일관성 검증 (WARN/FAIL)

- [ ] 컨셉의 성격 ↔ 프롬프트의 말투/행동: 모순 시 **FAIL**
- [ ] 컨셉의 세계관 ↔ 로어북의 설정: 모순 시 **FAIL**
- [ ] first_mes가 description·scenario 설정에 부합하는가
- [ ] 이미지 scene_key가 캐릭터·맥락에 맞는가
- [ ] 호칭이 관계 설정과 일치하는가 (예: 누나/언니/선배 혼용)

## F. 캐릭터 품질 (WARN) — 가벼운 웹소설 디폴트 기준

- [ ] 캐릭터 능동성이 호감도 단계별로 설계되어 있는가
- [ ] 비밀 보호가 **조건 기반 트리거**인가 ("절대 말하지 마" 방식은 WARN)
- [ ] 서술 품질 강화 지침이 있는가
- [ ] {{user}} 행동 제어 규칙이 있는가
- [ ] **욕망·태도·말투 핵이 선명한가** (흐릿해서 평균값으로 수렴하면 WARN — 평균회귀)
- [ ] **심리 설정이 과잉이라 캐릭터가 자기 내면을 해설하듯 경직되지 않는가** (과잉이면 WARN)
- [ ] **성격 ↔ 실제 말투/행동이 일관되는가** (어긋나면 FAIL)
- [ ] 캐릭터 간 욕망·말투 색깔이 차별화되는가 (겹치면 WARN)
  > 주: 모순·입체성은 가벼운 웹소설 디폴트에서 의무가 아니다. 컨셉이 명시적으로 심리·드라마를 요구할 때만 평가 (D/P 예외 판정).

## G. NSFW 톤·용어 (WARN/FAIL)

- [ ] 성적 용어 로어가 있는가 (현대물 NSFW: `보지/클리토리스/자지`, `constant:1 priority:95`) — 없으면 FAIL
- [ ] 사극/무협이면 용어 세트가 맞는가 (클리토리스 금지)
- [ ] 행위 묘사가 캐릭터 성격을 반영하는가 (천편일률이면 WARN)
- [ ] 안전 가드와 묘사 강도의 균형 — 과도한 가드로 묘사가 막히거나, 반대로 가드 부재

## H. 대화 시뮬레이션 (선택, 가능하면 수행)

3~5턴 시뮬레이션하여:
- [ ] first_mes가 자연스러운 진입을 유도하는가
- [ ] 로어북이 적절한 타이밍에 트리거되는가 (키워드 매칭 추적)
- [ ] 스테이터스/선택지가 일관되게 출력되는가
- [ ] 캐릭터가 설정대로 행동하는가
- [ ] 페이즈 시스템이 있으면 전환 조건이 실제로 작동하는가

실제 채팅 로그가 있으면(`GET /api/sessions/{id}/messages`) 분석하여 AI가 설정을 무시하는 패턴, 로어북 미트리거, 이미지 오매칭을 식별한다.

## 판정 기준

| 등급 | 의미 | 조건 |
|------|------|------|
| **PASS** | 등록/유지 가능 | FAIL 0개, WARN 3개 이하 |
| **WARN** | 가능하나 개선 권고 | FAIL 0개, WARN 4개 이상 |
| **FAIL** | 수정 필수 | FAIL 1개 이상 |

## 보고 형식

```markdown
## QA 체크리스트 결과: {캐릭터 이름}
### 종합: PASS / WARN / FAIL

| 영역 | 판정 | 비고 |
|------|------|------|
| A 변수 | PASS | |
| B 구조 | WARN | post_history 7줄 |
| ... | | |

### FAIL (수정 필수)
1. [영역] 문제 → **현재**: {발췌} → **수정안**: {구체안}

### WARN (개선 권고)
1. [영역] 권고 내용
```
