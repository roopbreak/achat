# AChat 엔진 레퍼런스

AChat 인터랙티브 픽션 엔진의 동작 원리. 스토리 작성·검수 페르소나가 공용으로 참조한다.
프롬프트·로어북이 엔진과 충돌하지 않으려면 이 문서의 제약을 지킨다.

> 진실의 원천은 `lib/context-builder.mjs`다. 이 문서와 코드가 다르면 코드가 맞다.

## 1. 컨텍스트 빌드 순서

매 턴 system blocks + messages 배열이 이렇게 조립된다:

```
[캐시됨] block 1: 서술 규칙 (NARRATION_RULES — 엔진 전역)
[캐시됨] block 2: 캐릭터 정보 (description + personality + scenario + 페르소나)
[캐시됨] block 3: 상시 로어북 (constant=1) + 이미지 카탈로그
[매턴 갱신] block 4: 유저 노트
[매턴 갱신] block 5: 동적 컨텍스트 (키워드 로어 + HypaMemory 벡터검색 + 세션 요약)
[매턴 갱신] block 6: post_history_instructions (매 턴 규칙 재주입)
[매턴 갱신] 최근 N턴 대화 (토큰 안전장치로 동적 조절, 최소 5턴 보장)
[매턴 갱신] 현재 유저 입력
```

설계 함의:
- **block 1~3은 캐시된다** → 안정적인 내용만. 자주 변하는 정보를 description에 넣으면 캐시 미스로 매 턴 토큰 낭비.
- **동적으로 변해야 하는 정보는 로어북·post_history로** 빼야 캐시가 유지된다.
- 유저 노트가 바뀌어도 block 1~3 캐시는 유지된다.
- post_history_instructions는 매 턴 마지막에 주입 → 핵심 규칙 재확인용. 짧고 핵심적으로(3~5줄 권장).

## 2. 프롬프트 작성 4원칙

| 원칙 | 내용 |
|------|------|
| **구체성** | 모호한 표현 금지. "아름다운 문체" → "캐릭터의 동작·표정·시선·자세를 구체적 디테일로 묘사" |
| **정보-지시 분리** | 캐릭터 정보 섹션과 출력 지시 섹션을 분리. 뒤섞이면 AI가 정보를 지시로 오해 |
| **과도한 지시 회피** | 동시 시스템 2개 이내. 체력·마나 실시간 계산 + 이미지 + 대사 동시 관리 금지 |
| **구조화** | 마크다운(`##`, `-`, `**`)으로 위계 표현. JSON 금지 |

## 3. 필드별 역할

| 필드 | 역할 | 캐시 | 길이 가이드 |
|------|------|------|------------|
| `description` | 캐릭터 상세설정 (정보-지시 분리 구조) | O | 3,000자 이내 |
| `personality` | 성격 1~2줄 요약. 메인 캐릭터 중심 | O | 짧게 |
| `scenario` | 시나리오/세계관 설정 | O | — |
| `first_mes` | 첫 대사. 캐릭터 성격이 드러나고 진입을 유도 | — | — |
| `post_history_instructions` | 매 턴 재주입 지시. 상태 연속성·상시 규칙·페이즈 | 매턴 | 3~5줄 권장 |

`description` 권장 구조:
```
## 캐릭터 정보   (이름·나이·직업·외모·성격·말투·배경)
## 관계          ({{user}}와의 관계, 주요 NPC)
## 행동 규칙     (능동성, 금지 행위, 비밀 보호)
## 출력 지시     (서술 방식, 스테이터스, 선택지)
```

변수: `{{char}}`(char_name 치환) / `{{user}}`(페르소나 이름 치환) — **이중 중괄호 필수**. `{char}` 단일 괄호는 치환 안 됨. description·로어북 본문에서는 캐릭터 실제 이름 직접 사용이 더 안전.

서술 시점 기본값: **{{user}} 1인칭**. 캐릭터 내면/속마음은 서술에 직접 쓰지 않고 표정·행동·말투·신체 반응으로 표현.

## 4. 서술 명령어

유저 입력의 서술 명령어를 엔진이 해석한다 (NARRATION_RULES 정의):
- `~행동~` — 행동 지시
- `~행동~+` — 능동성 강화 (최근 추가)
- `~~` — 이어쓰기
- `~~~` — 장면 전환
- `~!` — 서술자 모드
- `~~!` — 장면 전환 + 서술자 모드

프롬프트에서 이 명령어를 재정의하거나 충돌하는 지시를 넣지 않는다.

## 5. 로어북 동작

| 유형 | 설정 | 동작 |
|------|------|------|
| **상시 로어** | `constant: 1` | 항상 block 3에 주입. 캐시됨. 수치 규칙·핵심 세계관·서술 톤·성적 용어 정의용. 1~2개 권장, 4개+ 과다 |
| **키워드 로어** | `constant: 0` | scan_depth(기본 4턴) + 현재 입력에서 키워드 매칭 시 활성. NPC·장소·이벤트·비밀용 |

키워드 규칙:
- AND: `"키1+키2"` (둘 다 포함 시 매칭)
- NOT: `"-제외어"` (포함 시 매칭 차단)
- 명령어 트리거: `"!command"` (유저가 `!command` 입력 시 활성, `scan_depth: 1` 권장)
- 일상어·범용어 금지("아버지", "학교"). 명사형 키워드. 항목당 5개 이내.
- scan_depth: 기본 4, 명령어 트리거 1, 장기 맥락 8~10.

우선순위: `priority` 높은 순 → `insertion_order` 순. 토큰 예산 내에서만 활성.

**성적 용어 로어**(현대물 NSFW 필수, `constant: 1`, `priority: 95`):
- 현대물: `보지/클리토리스/자지` (음부·음핵·음경 표현 금지)
- 사극/무협: `보지/음핵/자지` (클리토리스 금지, 음문·남근 허용)

## 6. 이미지 시스템

- `story_images` 테이블: `charDir/sceneKey → filename` 매핑.
- 엔진이 시스템 프롬프트에 이미지 카탈로그를 주입 → AI가 마크다운 이미지로 응답에 삽입.
- 경로: `/images/{storyName}/{charDir}/{sceneKey}`
- **이미지 키워드 방식(`img: 표정`) 금지** — AChat은 scene_key 시스템. 내부/외부 이미지 혼용 금지.
- composition은 코어 55장(expression 15 + interaction 5 + adult 35) + customScenes(36~46장) 머지.

## 7. 스테이터스 시스템

기본 4종(권장):
```
📍 장소: (현재 장소)
👗 복장: (착장 + 상태)
🎬 상황: (장면 맥락)
💭 속마음: "(내면 독백)"
```
- 수치 스탯(선택): **1000 스케일** 사용 (0~100은 AI가 급변시킴). "화폐" 개념으로 선언 → 신중 변동. 평시 ±10~30, 극적 ±50, ±100 이상 금지.
- 선택지(선택): ①② 상황 맞는 행동(서술체, 따옴표 금지), ③ 항상 "자유 입력" 고정.

## 8. 페이즈 시스템

전개가 단계별로 구분되면 `post_history_instructions`에 페이즈를 설계한다. AI가 전개를 급하게 몰거나 설정을 무시하는 문제를 막는 가장 효과적인 구조.

**적용 기준** (하나 이상 해당 시 권장):
- 정체/비밀 공개 타이밍이 중요 (익명→발각)
- 관계 단계가 명확히 구분 (프로→진심)
- 멀티 캐릭터 등장 순서 제어 필요
- 에피소드 구조

**설계 원칙**:
1. 전환 조건은 **유저 행동 기반** — "유저가 영상을 수락했을 때" 등 AI가 판단 가능한 트리거만.
2. **턴 수 기반 조건 절대 금지** — AI는 턴을 세지 못한다.
3. 스테이터스에 **현재 Phase 표시 필수**.
4. Phase 수 3~4개 이내.

작성 형식:
```
[페이즈 시스템]
현재 Phase를 스테이터스에 반드시 표시. 전환 조건 미충족 시 다음 Phase로 넘어가지 않는다.
Phase 1 — {단계명}: {규칙/제약} / 전환 조건: {유저 행동 트리거} → Phase 2
Phase 2 — {단계명}: ...
Phase 3 — {단계명}: ...
```

단일 캐릭터 + 단순 관계 발전형은 페이즈 불필요 — 전역 NARRATION_RULES의 "분위기 점프 금지"로 충분.

## 9. 캐릭터 능동성 / 비밀 보호 / user 행동 제어

**능동성** (호감도 연동 단계별):
- 저호감: 무의식적 접근 (옆자리 앉기, 핑계로 말 걸기)
- 중호감: 의식적 시도 (선물·데이트 유도, 질투)
- 고호감: 적극적 표현 (먼저 스킨십, 직접 질문)

**비밀 보호**: "절대 말하지 마" 방식은 효과 없음. **조건 기반 트리거** — "오직 'XX'라는 단어가 언급될 때만 비밀을 이야기한다".

**{{user}} 행동 제어 (필수)**: {{user}}의 행동·대사·감정·판단을 임의로 결정하지 않는다. {{user}}가 선택하지 않은 행동을 미리 진행하지 않는다.

## 10. 멀티 캐릭터

- `{{char}}`는 **메인 캐릭터에만** 사용. 서브캐릭터는 직접 이름 명시.
- 서브캐릭터 상세 정보는 **로어북에 분산** (각 NPC 이름이 키워드).
- "장면에 있을 때만 등장" 원칙을 명시.

## 11. 금지 사항 체크

- ❌ 출력 분량 직접 지정 ("500자 이상") — 시스템이 maxTokens로 제어
- ❌ JSON 포맷 사용
- ❌ 이미지 키워드 방식 (`img: 표정`)
- ❌ 내부/외부 이미지 혼용
- ❌ description에 자주 변하는 내용 (캐시 미스)
- ❌ 턴 수 기반 조건
- ❌ 동시 시스템 3개 이상
- ❌ `{char}`/`{user}` 단일 괄호
- ❌ 상시 로어 4개 이상
